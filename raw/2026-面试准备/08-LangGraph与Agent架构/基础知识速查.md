## 细节

### [初] AI Agent 核心概念

**LLM（Large Language Model）**
- 基于海量文本预训练，具备理解和生成能力
- GPT-4、Claude、通义千问都是 LLM
- 核心能力：文本生成、CoT 推理、Tool Calling、上下文学习

**Agent = LLM + 工具 + 记忆 + 规划**
```
Agent = LLM（大脑）+ Tools（手）+ Memory（记忆）+ Planner（规划器）
```

**ReAct 模式**（Reasoning + Acting）
```
Thought → Action → Observation → Thought → Action → ...
思考要做什么 → 执行工具 → 观察结果 → 继续推理 → ...
```

**Token**：LLM 处理文本的最小单位。中文约 1-2 字/Token，英文约 0.75 词/Token。API 按 Token 计费。

### [初] LangChain 组件

| 组件 | 作用 | 示例 |
|------|------|------|
| Model I/O | 统一调用各种 LLM | `ChatOpenAI` |
| Prompt Template | 动态拼接 prompt | `PromptTemplate.from_template` |
| Chain | 串联多个 LLM 调用 | `LLMChain` |
| Agent | LLM + 工具编排 | `initialize_agent` |
| Memory | 对话历史存储 | `ConversationBufferMemory` |
| Tool | 外部能力 | `@tool` 装饰器 |
| Index | 文档检索 | `VectorStoreRetriever` |

**LangChain vs LangGraph**
- LangChain（LEL）：线性链，无循环无状态，适合简单 RAG
- LangGraph：有向图，支持循环、条件分支、人工中断、多轮记忆

### [中] LangGraph 核心

**StateGraph**
```python
class AlertState(TypedDict):
    messages: list
    alert_id: str
    severity: str

graph = StateGraph(AlertState)
graph.add_node("triage", triage_node)
graph.add_edge("__root__", "triage")
graph.add_edge("triage", END)
app = graph.compile()
```

**Edge 类型**
- 普通边：`graph.add_edge("node_a", "node_b")`
- 条件边：`graph.add_conditional_edges("node_a", route_fn, {...})`

**conditional_edges 示例**
```python
def route_after_triage(state):
    if state["triage_result"] == "true_positive":
        return "escalate"
    elif state["loop_count"] >= 3:
        return "auto_close"
    return "enrich"
```

### [中] Tool Calling

**@tool 装饰器**
```python
from langchain_core.tools import tool

@tool
def lookup_ip_reputation(ip: str) -> dict:
    """查询IP威胁情报"""
    return {"score": 85, "tags": ["malware-c2"]}
```

**bind_tools + tool_choice**
```python
model = ChatOpenAI(model="gpt-4o").bind_tools(tools)
# tool_choice: None（模型自主）/ "required"（至少调用一个）/ 指定工具名
```

### [高] Checkpointer

**持久化存储后端**
- `MemorySaver()`：内存，开发/测试用
- `SqliteSaver.from_conn_string("...")`：生产环境

**thread_id**
```python
config = {"configurable": {"thread_id": "alert-session-001"}}
result = app.invoke({"alert": alert}, config)
# 同一 thread_id 共享状态，LangGraph 自动从 checkpoint 恢复
```

**内部机制**：每次节点执行完成后，LangGraph 将完整 state 快照写入存储，包含 `messages` 追加历史。

### [高] Human-in-the-loop

**interrupt_before/after**
```python
compiled = app.compile(
    checkpointer=checkpointer,
    interrupt_before=["escalate"]  # 在 escalate 前挂起
)
# 执行到 escalate 前暂停，外部读取状态展示给分析师
# 分析师决策后再次 invoke，LangGraph 从断点恢复
```

**断点恢复**
```python
result = app.invoke(
    {"human_decision": "approved", "messages": ["确认升级"]},
    config  # 同一 thread_id
)
```

### [高] 生产调优

**Prompt 版本化**
```python
PROMPTS = {
    "v1.0": """你是一个SOC分析师。分析以下告警...""",
    "v1.1": """你是一个SOC分析师，具备MITRE ATT&CK知识...""",
}
def get_triage_prompt(version="v1.1"):
    return PROMPTS.get(version, PROMPTS["v1.1"])
```

**Pydantic Schema Validation**
```python
class TriageOutput(BaseModel):
    severity: str
    attack_stage: str
    confidence: float
structured_model = model.with_structured_output(TriageOutput)
```

**Token 预算控制**：`messages[-N:]` 截断历史，防止上下文膨胀超过模型 token 限制。
