## 细节

### [初] Docker 基本概念

**镜像/容器**
- 镜像（Image）：只读模板
- 容器（Container）：镜像的运行实例

**基础命令**
```bash
docker pull python:3.11-slim
docker build -t myapp:1.0 .
docker run -d -p 8080:8000 --name myapp myapp:1.0
docker ps / docker ps -a
docker logs -f --tail 100 myapp
docker exec -it myapp /bin/bash
docker stats
```

### [中] Dockerfile 指令

| 指令 | 作用 | 备注 |
|------|------|------|
| FROM | 指定基础镜像 | 第一条指令 |
| RUN | 构建时执行命令 | 合并减少层数 |
| COPY | 复制文件（优先）| |
| ADD | 复制+自动解压/URL下载 | 不推荐 |
| EXPOSE | 声明端口 | 文档作用 |
| CMD | 容器启动默认命令 | 可被覆盖 |
| ENTRYPOINT | 固定入口 | 不可被覆盖 |

**CMD vs ENTRYPOINT**：CMD 定义默认参数，可被 docker run 覆盖；ENTRYPOINT 定义不可变入口，参数追加到其后

### [中] 多阶段构建

```dockerfile
# Stage 1: 构建
FROM golang:1.22 AS builder
WORKDIR /app
RUN go build -o myapp

# Stage 2: 运行
FROM alpine:3.19
COPY --from=builder /app/myapp .
CMD ["./myapp"]
```

**.dockerignore**
```
node_modules/
.git/
.env
*.log
tests/
__pycache__/
```

### [高] Docker 网络

**四种模式**
- bridge（默认）：连 docker0 网桥，NAT 通信，需 `-p` 映射
- host：直接用宿主机网络栈，性能好但端口冲突
- none：只有 loopback
- overlay：Docker Swarm 跨主机容器通信

**自定义 bridge**：内置 DNS，容器间可用服务名直接访问

### [高] 数据持久化

| 类型 | 存储位置 | 生命周期 | 适用场景 |
|------|---------|---------|---------|
| Volume | /var/lib/docker/volumes | 独立于容器 | 数据库 |
| Bind Mount | 宿主机任意路径 | 依赖宿主机 | 开发热更新 |
| tmpfs Mount | 内存 | 容器停止即消失 | 敏感数据 |

`docker rm` 不删除 Volume，需单独 `docker volume rm`

### [高] K8s 核心对象

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: my-app
  template:
    spec:
      containers:
        - name: app
          image: myapp:v1
          ports:
            - containerPort: 8080
          readinessProbe:
            httpGet:
              path: /health
              port: 8080
          livenessProbe:
            tcpSocket:
              port: 8080
```

---

### [高] K8s 探针

| 探针 | 作用 | 失败后果 |
|------|------|---------|
| readinessProbe | 就绪探针 | 流量不进入 Pod |
| livenessProbe | 存活探针 | 重启 Pod |
| startupProbe | 启动探针 | 启动完成前禁用其他探针 |
