### 响应式原理
```
Vue 2 和 Vue 3 的响应式原理各自是怎么工作的？Object.defineProperty 和 Proxy 各自的优缺点？Vue 3 为什么改用 Proxy？
追问：
- Vue 2 中数组的响应式处理有什么特殊之处？
- Vue 3 中 ref 和 reactive 有什么区别？
```
**[10秒结论]** Vue 2 用 `Object.defineProperty` 劫持 getter/setter 实现响应式，数组需特殊处理；Vue 3 用 Proxy 深层代理整个对象，性能更好且支持更多数据类型。

**[90秒拆解]**
- **Vue 2 defineProperty**：
  - 在 data 初始化时递归遍历所有属性，用 Object.defineProperty 绑定 getter（收集依赖）和 setter（触发更新）
  - 无法检测新增属性（需 Vue.set）和删除属性（需 Vue.delete）
  - **数组特殊处理**：Vue 2 覆盖了数组的 push/pop/splice 等方法，在调用这些方法时触发更新；直接通过索引赋值无法触发（需 Vue.set）
- **Vue 3 Proxy**：
  - Proxy 代理整个对象而非每个属性，性能更好
  - 支持新增/删除属性的自动检测（通过 `has`、`deleteProperty` 拦截）
  - 支持 Map、Set、Class 等数据类型
  - `ref` 包装基本类型，`reactive` 深层代理对象
- **Proxy 优缺点**：优势是性能好、功能全；缺点是不支持 IE11

**[20秒收尾]** Vue 3 的响应式系统在 track（收集依赖）和 trigger（触发更新）两个函数中实现，比 Vue 2 更透明和可扩展。

### 生命周期
```
Vue 2 和 Vue 3 的生命周期钩子各自有哪些？setup 中怎么使用生命周期钩子？keep-alive 的 activated 和 deactivated 是什么时候触发？
追问：
- created 和 mounted 的区别是什么？
- onMounted 里能获取到 DOM 吗？
```
**[10秒结论]** Vue 2/3 生命周期一致：创建（beforeCreate/created → beforeMount/mounted）→ 更新（beforeUpdate/updated）→ 销毁（beforeUnmount/unmounted）；setup 在 beforeCreate 之前执行；keep-alive 的 activated 在组件激活时触发。

**[90秒拆解]**
- **创建阶段**：beforeCreate（data/methods 未初始化）→ created（data/methods 已就绪，DOM 不可用）→ beforeMount（模板编译完成）→ mounted（DOM 已挂载，可操作 $el）
- **更新阶段**：beforeUpdate（数据变化，DOM 更新前）→ updated（DOM 更新后，避免在此修改数据引起循环）
- **销毁阶段**：beforeUnmount（组件卸载前，清理定时器/订阅）→ unmounted（完全卸载）
- **keep-alive**：include/exclude 缓存动态组件，切换时触发 activated（激活）和 deactivated（停用）钩子
- **setup 中的生命周期**：通过 `onMounted(() => {})`、`onUnmounted(() => {})` 等函数调用，对应 Vue 2 的 created/mounted 等钩子

**[20秒收尾]** created 中可以发起 Ajax 请求，但 DOM 未生成不适合操作；mounted 是最早能操作 DOM 的钩子，适合做图表初始化。

### 组件通信
```
父子组件之间 props/$emit 怎么用？$attrs 和 $listeners 是什么？provide/inject 适合什么场景？Vuex/Pinia 怎么选？
追问：
- v-model 在组件上是怎么工作的？
- 兄弟组件通信除了 Vuex/Pinia 还能怎么实现？
```
**[10秒结论]** props 向下传递数据和回调，$emit 向上发送事件；$attrs 包含未在 props 声明的属性，$listeners 包含所有父组件监听的事件；provide/inject 适合深层嵌套传值；Pinia 替代 Vuex 是 2023 年后的主流选择。

**[90秒拆解]**
- **props/$emit**：父组件通过 props 传数据，子组件通过 $emit 触发父组件方法（如 `this.$emit('update', value)`）
- **v-model 原理**：`<input v-model="msg">` 等价于 `:value="msg" @input="msg = $event.target.value"`；在组件上 `v-model="title"` 等价于 `:modelValue="title" @update:modelValue="title = $event"`
- **$attrs/$listeners**：父组件传的非 prop 属性（class/style/id）在 $attrs 中，事件在 $listeners 中。Vue 3 合并为 $attrs（包含属性和事件）
- **provide/inject**：祖先组件 provide，后代组件 inject，无需逐层 props drilling，适合插件和组件库
- **Pinia vs Vuex**：Pinia API 更简洁（无需 mutation），原生支持 TypeScript，DevTools 集成更好，推荐新项目使用 Pinia

**[20秒收尾]** SOC 平台用 Pinia 管理告警列表状态（多组件共享），简单的弹窗状态用 provide/inject，深层嵌套的 form 数据用 props + emit。

### Vue 3 组合式 API
```
setup 函数是什么？Composition API 相比 Options API 有什么优势？watch 和 watchEffect 有什么区别？computed 和 methods 的区别？
追问：
- setup 中怎么获取组件实例？
- toRefs 解决了什么问题？
```
**[10秒结论]** setup 是 Vue 3 组件的入口函数，在 beforeCreate 之前执行；Composition API 将同一逻辑相关代码放在一起（逻辑复用优于 Options API 按选项分离）；watch 显式指定监听目标，watchEffect 自动收集依赖。

**[90秒拆解]**
- **setup 特性**：return 的对象可在模板中使用；setup 参数 (props, context) 可以解构（需用 toRefs 保持响应式）
- **Options API vs Composition API**：
  - Options API：data/methods/computed/watch 按选项组织，逻辑分散在不同选项
  - Composition API：相关逻辑（变量+方法+计算+副作用）放在一起，更易维护和复用
- **watch vs watchEffect**：
  - watch：显式指定监听目标，懒执行，变化时执行回调；可选 immediate/flush/deep 选项
  - watchEffect：自动收集依赖，立即执行；适合不需要旧值、不关心何时停止的场景
- **computed vs methods**：computed 有缓存（依赖不变不重算），methods 每次调用都重新执行
- **toRefs**：将 reactive 对象解构为 ref，保持响应式：
  ```javascript
  const state = reactive({ count: 0, name: 'vue' });
  const { count, name } = toRefs(state);  // count/name 变为 ref
  ```

**[20秒收尾]** Vue 3 的 `<script setup>` 语法糖让 setup 更简洁，编译器自动处理 import/ref/withDefaults，代码量减少 50%。

### 路由与状态管理
```
Vue Router 的 hash 模式和 history 模式各自原理是什么？怎么实现导航守卫（beforeEach/beforeResolve/afterEach）？Vue Router 4 的动态路由怎么用？
追问：
- 路由懒加载怎么写？
- router.push 和 router.replace 有什么区别？
```
**[10秒结论]** hash 模式用 URL # 锚点变化不触发页面刷新，history 模式用 HTML5 History API（pushState/replaceState）需要服务器配合；导航守卫在路由切换前后做权限验证和逻辑处理。

**[90秒拆解]**
- **hash vs history**：
  - hash：`window.location.hash` 变化不触发页面刷新，兼容性更好，但 URL 不美观
  - history：`history.pushState/replaceState/popState` 实现，URL 干净，但直接访问子路径需要服务器重定向（否则 404）
- **导航守卫**：
  - beforeEach：全局前置守卫，每个路由跳转前执行（适合权限验证）
  - beforeResolve：全局解析守卫，在导航确认前、所有组件守卫和异步路由组件解析后调用
  - afterEach：全局后置钩子，导航完成后调用（适合页面埋点）
- **动态路由**：
  ```javascript
  router.addRoute({ path: '/admin', component: Admin, beforeEnter: guard })
  router.removeRoute('admin')
  ```
- **路由懒加载**：
  ```javascript
  const Dashboard = () => import('./views/Dashboard.vue')
  // 或
  const Dashboard = () => import(/* webpackChunkName: "dashboard" */ './views/Dashboard.vue')
  ```

**[20秒收尾]** 生产环境用 history 模式 + 动态加载路由表，首页白屏时间可控制在 1s 以内。

### 虚拟 DOM 与渲染
```
Vue 的虚拟 DOM 是什么？template 是怎么编译成 render 函数的？Vue 3 的 PatchFlag 静态标记是什么？是怎么优化渲染性能的？
追问：
- v-if 和 v-show 的区别？各自适用什么场景？
- Vue 的模板编译是在构建时还是运行时？
```
**[10秒结论]** Vue 的 template 编译成 render 函数，render 返回虚拟 DOM 树，patch 时对比新旧虚拟 DOM 找差异并更新；Vue 3 的 PatchFlag 在编译时标记动态节点，跳过静态子树的 diff。

**[90秒拆解]**
- **template → render**：Vue 单文件组件的 template 在构建时由 vue-loader 编译成 render 函数，运行时不额外解析 template
- **虚拟 DOM 节点（VNode）**：JS 对象，描述真实 DOM 的类型（div/span）、属性（attrs/props）和子节点
- **PatchFlag 优化**：静态节点标记为 1（不需要对比），只有动态属性（如 `:class`、`{{name}}`）被标记为 2/3 等，diff 时只比较这些标记位：
  ```javascript
  h('div', { class: 'static', [PatchFlags.CLASS]: dynamicClass }, children)
  ```
- **v-if vs v-show**：v-if（false）不渲染 DOM，v-show 用 `display:none` 控制显示；频繁切换用 v-show，初次渲染条件为 false 用 v-if
- **模板编译时机**：构建时（webpack vue-loader），运行时（无构建步骤用 template 字符串）

**[20秒收尾]** Vue 3 的静态提升（hoistStatic）将静态节点提到渲染函数外，避免重复创建，大幅减少内存分配。

### Vue 3 新特性
```
Teleport 是什么？Suspense 适合什么场景？Vue 3 的 Tamplate Slots 和 Vue 2 的 Slot 有什么变化？CSS v-bind 在 style 标签里怎么用？
追问：
- Vue 3 的 emit 是怎么实现类型检查的？
- defineProps 和 defineEmits 是什么？
```
**[10秒结论]** Teleport 将组件 DOM 传送到任意位置（常见于 modal）；Suspense 用于异步组件加载完成前的占位；Vue 3 Slot 用 v-slot 统一语法，defineProps/defineEmits 是编译器宏（无需 import）。

**[90秒拆解]**
- **Teleport**：
  ```vue
  <Teleport to="body">
    <Modal v-if="show" />
  </Teleport>
  ```
  组件逻辑位置不变，DOM 渲染到 body 下，解决 z-index 和 overflow 问题
- **Suspense**：配合异步组件使用，展示加载状态：
  ```vue
  <Suspense>
    <template #default><AsyncComponent /></template>
    <template #fallback><Loading /></template>
  </Suspense>
  ```
- **Slot 变化**：Vue 2 作用域 slot 有 `slot="header"` 和 `slot-scope`；Vue 3 统一为 `v-slot:header`，更简洁
- **CSS v-bind**：
  ```vue
  <style scoped>
  .text { color: v-bind(color); }
  </style>
  ```
  JS 中的响应式变量直接用于 CSS
- **defineProps/defineEmits**：编译器宏，只能在 `<script setup>` 中使用，自动导出类型（运行时可不写类型）：
  ```vue
  <script setup>
  const props = defineProps<{ title: string; count?: number }>()
  const emit = defineEmits<{ (e: 'update', val: number): void }>()
  </script>
  ```

**[20秒收尾]** Vue 3.4 的 `defineModel` 让双向绑定更简单，`v-model:title="value"` 自动对应 props 和 emit，无需手动定义。

### 性能与工程化
```
Vue 项目怎么优化首屏加载？Vue.use 和 Vue.component 是什么？Vue 的依赖收集机制是怎么工作的？SSR（Nuxt）和 SSG 各自适用什么场景？
追问：
- Vue 的异步组件怎么写？
- Vue.config.errorHandler 有什么用？
```
**[10秒结论]** 首屏优化：路由懒加载 + 组件异步加载 + SSR；Vue.use 注册插件，Vue.component 注册全局组件；Vue 的依赖收集在 getter 中建立 watcher 和 dep 的关联。

**[90秒拆解]**
- **首屏加载优化**：
  - 路由懒加载：`() => import('./Foo.vue')`
  - 组件异步加载：`defineAsyncComponent`
  - 预加载：`<link rel="preload">` + webpackChunkName
  - 图片懒加载：vue-lazyload
- **Vue.use/Vue.component**：Vue.use(Plugin) 调用插件的 install 方法并阻止重复安装；Vue.component 注册全局组件，使所有子组件可直接使用（不用 import）
- **依赖收集机制**：data 属性的 getter 中调用 `dep.depend()` 建立依赖，setter 中调用 `dep.notify()` 通知所有 watcher 更新
- **Nuxt SSR vs SSG**：SSR（服务端渲染）适合内容个性化（SEO + 用户定制）；SSG（静态站点生成）适合内容固定（文档、博客），构建时生成 HTML，性能最高
- **异步组件**：
  ```javascript
  defineAsyncComponent({
    loader: () => import('./Foo.vue'),
    loadingComponent: Loading,
    errorComponent: Error,
    delay: 200
  })
  ```

**[20秒收尾]** Vue 项目必配 vue.config.js 配置 vue-loader 的 productionSourceMap（生产关闭）、chainWebpack 做代码分割和别名配置。
