## 细节

### [初] 核心概念

**Vue 2 响应式原理（Object.defineProperty）**
- data 初始化时递归遍历属性，用 defineProperty 绑定 getter（收集依赖）和 setter（触发更新）
- 无法检测新增属性（需 Vue.set）和删除属性（需 Vue.delete）
- 数组用覆盖方法（push/pop/splice）触发更新，直接索引赋值不触发

**Vue 3 响应式原理（Proxy）**
- Proxy 代理整个对象，深层代理，get 时收集依赖，set 时触发更新
- 支持新增/删除属性检测，支持 Map/Set/Class
- `ref` 包装基本类型，`reactive` 深层代理对象

**模板语法**
```vue
<template>
  <div :class="isActive ? 'active' : ''">
    {{ message }}
    <button @click="handleClick">点击</button>
  </div>
</template>
```

### [初] 生命周期

| 钩子 | 说明 |
|------|------|
| beforeCreate | data/methods 未初始化 |
| created | data/methods 就绪，DOM 不可用，适合发起请求 |
| beforeMount | 模板编译完成 |
| mounted | DOM 已挂载，可操作 $el，适合初始化图表 |
| beforeUpdate | 数据变化，DOM 更新前 |
| updated | DOM 更新后，避免在此修改数据 |
| beforeUnmount | 组件卸载前，清理定时器/订阅 |
| unmounted | 完全卸载 |

**keep-alive**：`activated`（激活）和 `deactivated`（停用）钩子

### [中] 组件通信

**props/$emit**
```vue
// 父
<Child :msg="msg" @update="msg = $event" />
// 子
props: ['msg']
this.$emit('update', newValue)
```

**v-model 原理**
```vue
<!-- <input v-model="msg"> 等价于 -->
<input :value="msg" @input="msg = $event.target.value" />
```

**$attrs/$listeners**
- Vue 3 合并为 `$attrs`（包含属性和事件）
- 用于透传 attrs 给子组件根元素

**provide/inject**：祖先 provide，后代 inject，适合深层嵌套传值

**Pinia**
- `state` 定义状态，`getters` 计算属性，`actions` 同步/异步方法
- 比 Vuex 更简洁，原生支持 TypeScript

### [中] Vue Router

**hash vs history**
- hash：`window.location.hash` 变化不触发页面刷新
- history：`history.pushState/replaceState`，需服务器配合重定向

**导航守卫**
```javascript
router.beforeEach((to, from, next) => { /* 全局前置 */ next() })
router.afterEach((to, from) => { /* 全局后置 */ })
```

**懒加载**
```javascript
const Dashboard = () => import('./views/Dashboard.vue')
```

---

### [中] Composition API

**setup 函数**
- 在 beforeCreate 之前执行
- return 的对象可在模板中使用
- `setup(props, context)` 可解构，但需用 toRefs 保持响应式

**ref vs reactive**
```javascript
const count = ref(0)       // ref，.value 访问
const state = reactive({    // reactive，整个对象响应式
  name: 'vue',
  count: 0
})
```

**computed vs methods**
- computed 有缓存（依赖不变不重算）
- methods 每次调用都重新执行

**watch vs watchEffect**
- watch：显式指定监听目标，懒执行，可选 immediate
- watchEffect：自动收集依赖，立即执行

### [高] 性能优化

**v-if vs v-show**
- v-if（false）：不渲染 DOM
- v-show（false）：`display:none`，始终在 DOM 中
- 频繁切换用 v-show，初次渲染条件为 false 用 v-if

**computed vs methods**：computed 有缓存，methods 无缓存

**v-memo**：`v-memo="[dep]"` 记忆值，dep 变化才重新渲染

**长列表优化**：虚拟滚动（vue-virtual-scroller），只渲染可见行

---

### [高] Vue 3 新特性

**Teleport**
```vue
<Teleport to="body">
  <Modal v-if="show" />
</Teleport>
```

**Suspense**：异步组件加载完成前的占位

**defineProps/defineEmits**
```vue
<script setup>
const props = defineProps<{ title: string; count?: number }>()
const emit = defineEmits<{ (e: 'update', val: number): void }>()
</script>
```

**CSS v-bind**
```vue
<style scoped>
.text { color: v-bind(color); }
</style>
```

---

### [高] SSR 与 Nuxt

**Nuxt vs Vue SSR**
- Nuxt 提供文件约定路由、SSR/SSG/ISR、开箱即用的 SEO
- Vue SSR 需要手动配置 webpack 和服务端渲染逻辑

**Hydration 问题**
- 服务端渲染的 HTML 与客户端 hydration 不匹配
- 用 `ClientOnly` 包裹动态组件避免 hydration 错误

**Nuxt 3 auto-import**
- 组件/组合式函数自动导入，无需 import
- `plugins/` 自动执行，`composables/` 自动注册
