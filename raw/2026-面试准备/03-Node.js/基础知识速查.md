## 细节

### [初] 事件循环

**六个阶段**
1. timers：执行 setTimeout/setInterval 回调
2. pending callbacks
3. idle/prepare（内部）
4. poll：获取新 I/O 事件
5. check：执行 setImmediate
6. close callbacks

**执行顺序**
```
process.nextTick（最高）→ 微任务（Promise.then）→ 宏任务（setTimeout/setImmediate）→ I/O 回调
```

**宏任务 vs 微任务**
- 微任务：在当前阶段结束后、下一个阶段开始前全部执行完
- 宏任务：在事件循环的对应阶段执行

**Libuv**：跨平台 I/O 抽象层，封装 epoll（Linux）/kqueue（macOS）

---

### [初] Promise

**三种状态**：pending（进行中）→ fulfilled（已成功）/ rejected（已失败），状态一旦改变不可逆

**链式调用原理**：`promise.then(onFulfilled)` 返回新 Promise，新 Promise 状态由返回值决定

**async/await 异常**
```javascript
try {
  const result = await fetchData();
} catch (error) {
  console.error('捕获到错误:', error);
}
```

### [中] 模块机制

**CJS vs ESM**

| 维度 | CommonJS | ES Module |
|------|-----------|-----------|
| 加载时机 | 运行时同步 | 编译时静态分析 |
| 导入语法 | require/exports | import/export |
| 值拷贝 | 值拷贝 | 值引用（动态只读绑定）|
| 动态 | 可条件加载 | 必须在模块顶层 |

**循环引用**：B 在 A 未加载完时返回已执行部分（undefined），JavaScript 以深度优先保证加载顺序

**exports vs module.exports**：`exports = {}` 切断引用导致 export 失效；`exports.xxx = val` 正确

---

### [中] Buffer 与 Stream

**Buffer**：`Buffer.from('hello')`、`Buffer.alloc(1024)`，处理 TCP 流、文件 I/O、加密等二进制数据

**Stream 四种类型**：Readable / Writable / Duplex / Transform

**背压**：Writable 写入速度跟不上 Readable 时，`write()` 返回 false，Readable 应暂停发送数据

**createReadStream vs readFile**
- `readFile`：一次性读入内存，大文件 OOM
- `createReadStream`：分块读取，内存占用稳定

---

### [高] V8 垃圾回收

**新生代**：Scavenge 算法，From/To 两半空间复制，50% 空间利用率

**老年代**：Mark-Sweep（标记清除）+ Mark-Compact（标记整理）压缩

**内存泄漏排查**：`node --inspect` 启动 Chrome DevTools，heapdump 定期生成快照对比

**常见泄漏**：全局变量累积、闭包引用、EventEmitter 监听未移除、定时器未清理

---

### [高] 性能优化

**cluster 模块**：主进程 fork 多个 Worker 进程，独立事件循环，IPC 通信，主进程监听端口分发请求

**pm2 零宕机**：SIGTERM 时先停止接收新请求，等现有请求处理完再退出；重载时先启动新进程再关闭旧进程

**child_process**：适合运行独立子进程（Python 脚本、FFmpeg），不共享内存，通信通过 stdin/stdout

---

### [高] 安全

**原型链污染**：用户输入控制 `__proto__` 或 `constructor.prototype`，可修改 Object 原型
- 防御：`Object.create(null)` 创建无原型对象；禁用用户输入控制原型属性

**eval 危险**：执行字符串为代码，攻击者可注入恶意脚本
- 替代：`new Function('param', 'return code')` 或模板引擎

**依赖漏洞**：`npm audit` 扫描 CVE；CI 集成 Snyk/Dependabot
