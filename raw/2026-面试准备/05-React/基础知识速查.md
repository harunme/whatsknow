## 细节

### [初] 核心概念

**虚拟 DOM**
- React 在内存维护两棵虚拟 DOM 树（current 和 workInProgress）
- diff 算法（同层对比 + key 匹配）找出最小更新集，批量 commit 到真实 DOM
- O(n) 复杂度策略：同层对比（不同则替换子树），列表用 key 保持稳定

**JSX 语法**
```jsx
const element = <h1 className="title">Hello, {name}</h1>;
// JSX 会被 Babel 编译为 React.createElement()
```

**单向数据流**
- 数据从父组件通过 props 向下传递
- 子组件通过调用父组件传来的回调函数触发更新
- 组件不能直接修改 props

**组件分类**
- 函数组件（推荐）：Hooks 实现状态和副作用
- 类组件：extends React.Component，有 state 和生命周期方法

---

### [初] 生命周期

**React 16.3+ 生命周期**

| 阶段 | 钩子 | 说明 |
|------|------|------|
| 挂载 | constructor → render → React.DOM 更新 → componentDidMount | 适合发起请求 |
| 更新 | shouldComponentUpdate → render → getSnapshotBeforeUpdate → React.DOM 更新 → componentDidUpdate | 适合 DOM 操作 |
| 卸载 | componentWillUnmount | 清理订阅/定时器 |

**React 16.3 前后变化**
- 废弃：`componentWillMount`（异步渲染不安全）、`componentWillReceiveProps`、`componentWillUpdate`
- 新增：`getDerivedStateFromProps`（挂载和更新时调用，返回对象合并到 state）、`getSnapshotBeforeUpdate`（DOM 更新前获取快照）

**Hooks 替代生命周期**
```javascript
useEffect(() => {
  // componentDidMount + componentDidUpdate
  return () => {}; // componentWillUnmount
}, [dep]); // 依赖数组控制时机
```

### [中] Hooks 核心

**useState**
```javascript
const [count, setCount] = useState(0);
// setCount(prev => prev + 1); // 函数式更新，避免闭包陷阱
```

**useEffect**
- 无依赖：每次 render 后执行
- `[]`：挂载后 + 卸载前执行
- `[dep]`：dep 变化后执行
- 清理函数在下一次 effect 执行前和卸载时运行

**useCallback / useMemo**
```javascript
const memoizedFn = useCallback(() => doSomething(a), [a]);
const memoizedValue = useMemo(() => computeExpensive(a), [a]);
```

**自定义 Hook**：`use` 开头，内部可调用其他 Hook，封装有状态逻辑

---

### [中] 状态管理

**Context API**
- `createContext` → Provider 提供值 → Consumer 或 `useContext` 消费
- value 变化时，所有消费该 Context 的组件强制更新（不比较引用）

**Redux 核心**
- 单一 store + pure reducer + action dispatch
- Redux Toolkit：`createSlice` 自动生成 action/reducer，内置 immer

**Zustand**
- 轻量（无 Provider、无 immutable 约束）
- 适合中小型项目状态管理

**React Query**
- 服务端状态管理（请求缓存、自动刷新、分页）
- `useQuery` / `useMutation`

---

### [中] 性能优化

**React.memo**
- 高阶组件，包装后 props 不变时不重新渲染
- 默认浅比较 props，自定义比较函数做深度比较（慎用，有性能开销）

**懒加载**
```javascript
const Dashboard = React.lazy(() => import('./Dashboard'));
<Suspense fallback={<Loading />}><Dashboard /></Suspense>
```

**虚拟滚动**：`react-window` 渲染可见区域节点（10000行 → 20个 DOM 节点）

**reconciliation 触发条件**
- state 变化（setState）
- props 变化（父组件重渲染）
- context 变化

---

### [高] 组件设计

**HOC**：接收组件返回新组件（如 `withRouter`、`connect`）
- 问题：wrapper 地狱、ref 需要 forwardRef 转发

**Render Props**：将 render 函数作为 props 传入复用逻辑
```jsx
<MouseRenderProps render={pos => <Cat mouse={pos} />} />
```

**ForwardRef**：`React.forwardRef((props, ref) => ...)` 转发 ref 到 DOM 元素

**Compound Components**：`Select` + `Option` 隐式共享 state

---

### [高] React 18 新特性

**Concurrent Mode**
- 渲染可中断，优先级高的更新可打断低优先级更新
- `useTransition`：标记非紧急更新
- `useDeferredValue`：值防抖

**Automatic Batching**
- React 18 前只有事件处理函数内 setState 批量
- 18 后 Promise/setTimeout/setInterval 内的 setState 也批量，减少重渲染

**Suspense SSR**
- 服务端流式 HTML + 客户端选择性水合
- 提升首屏加载体验

### [高] TypeScript 集成

```typescript
// Props 类型
interface ButtonProps { label: string; onClick: () => void; }
// useRef 泛型
const inputRef = useRef<HTMLInputElement>(null);
// 事件处理
const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => { ... }
// ForwardRef 泛型保留
const FancyButton = React.forwardRef<HTMLButtonElement, Props>((props, ref) => (
  <button ref={ref}>{props.label}</button>
));
```
