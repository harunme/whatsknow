## 答案

### A1: 核心概念
**[10秒结论]** React 单向数据流即 props 从父到子流动，组件不可直接修改 props；虚拟 DOM 是真实 DOM 的 JS 对象映射，diff 算法对比新旧虚拟 DOM 树，只对差异做最小化真实 DOM 操作。

**[90秒拆解]**
- **单向数据流**：数据从父组件通过 props 向下传递，子组件通过回调函数向上传递事件。组件不能直接修改 props，只能通过调用父组件传来的函数触发更新
- **虚拟 DOM 工作原理**：React 在内存维护两棵虚拟 DOM 树（current 和 workInProgress），diff 算法（同层对比 + key 匹配）找出最小更新集，最后批量 commit 到真实 DOM
- **为什么需要虚拟 DOM**：真实 DOM 操作昂贵（重排重绘），虚拟 DOM 通过批量更新 + 最小化 diff 将多次操作合并，减少真实 DOM 调用次数
- **diff 算法**：O(n) 复杂度策略——同层对比（不同则直接替换子树），同节点对比（tag/key 不同则替换，属性变化则更新），列表用 key 保持稳定位置

**[20秒收尾]** React 16 Fiber 将渲染工作拆成小单元（Fiber 节点），可中断可恢复，解决了大量更新时主线程阻塞问题（ANR）。

### A2: 生命周期
**[10秒结论]** 挂载（constructor→render→componentDidMount）、更新（props/state 变化→render→componentDidUpdate）、卸载（componentWillUnmount）；getDerivedStateFromProps 用于根据 props 派生 state，getSnapshotBeforeUpdate 用于在 DOM 更新前获取快照。

**[90秒拆解]**
- **挂载阶段**：constructor（初始化 state 和绑定 this）→ render（返回 JSX）→ React 更新 DOM → componentDidMount（副作用如请求数据、订阅）
- **更新阶段**：shouldComponentUpdate（决定是否重新渲染）→ render → getSnapshotBeforeUpdate（获取 DOM 滚动位置等）→ React 更新 DOM → componentDidUpdate
- **卸载阶段**：componentWillUnmount（清理订阅、定时器、取消请求）
- **getDerivedStateFromProps**：static 方法，每次 render 前调用，返回对象合并到 state 或 null。适用于 props 变化时同步更新内部 state（慎用，容易产生 derived state 陷阱）
- **getSnapshotBeforeUpdate**：在 DOM 更新前被调用，返回值传给 componentDidUpdate 的第三个参数。典型用法：聊天窗口滚动位置保持

**[20秒收尾]** Hooks 完全覆盖生命周期：useEffect(()=>{}, []) 等于 componentDidMount/WillUnmount；无依赖数组等于 componentDidUpdate。

### A3: Hooks
**[10秒结论]** useState 适合简单状态，useReducer 适合复杂状态逻辑；useEffect 依赖数组控制执行时机，空数组只在挂载/卸载运行；useCallback 缓存函数引用，useMemo 缓存计算值。

**[90秒拆解]**
- **useState vs useReducer**：useState 是 useReducer 的封装（dispatch = () => setState(prev => newState)）；useReducer 适合有多个子值或下一个状态依赖上一个状态的场景（如表单多个字段）
- **useEffect 依赖数组**：
  - 无依赖：每次 render 后执行（componentDidUpdate）
  - `[]`：只在挂载后 + 卸载前执行（componentDidMount + WillUnmount）
  - `[dep]`：dep 变化后执行
  - 清理函数在下一次 effect 执行前和卸载时运行
- **useCallback vs useMemo**：
  - useCallback：`const fn = useCallback(() => { doSomething(a) }, [a])`，缓存函数引用，避免子组件因函数引用变化而重新渲染
  - useMemo：`const val = useMemo(() => computeExpensive(a), [a])`，缓存昂贵计算结果
- **自定义 Hook**：以 `use` 开头的函数，内部可调用其他 Hook，用于复用有状态的逻辑（如 `useDebounce`、`useLocalStorage`）

**[20秒收尾]** 避免过度优化，useCallback/useMemo 本身有开销，只有在子组件用 React.memo 或列表渲染中才值得用。

### A4: 状态管理
**[10秒结论]** Context 通过 createContext + Provider/Consumer（或 useContext）实现跨层级数据传递；Redux 通过单一 store + pure reducer + action dispatch 管理全局状态；Redux Toolkit 用 createSlice 自动生成 action 和 reducer，集成了 immer 允许 "mutate" 语法。

**[90秒拆解]**
- **Context 原理**：createContext 返回包含 Provider 和 Consumer 的对象；Provider 的 value 变化时，所有消费该 Context 的组件强制更新（不比较引用，深层穿透传递）
- **Redux 核心流程**：store 持有单一状态树 → dispatch(action) → reducer(pure function) 根据 action type 返回新 state → subscriber 收到通知
- **Redux Toolkit（RTK）改进**：
  - `createSlice`：自动生成 action creators 和 action types
  - `createAsyncThunk`：处理 async 逻辑
  - 内置 immer，允许 `state.push()` / `state.xxx = val` 的可变写法
  - 默认集成 Redux DevTools 和redux-thunk
- **Zustand vs Redux**：Zustand 更轻量（无 provider、无 reducer 模板、无 immutable 约束），适合中小型项目；Redux 生态更成熟，适合大型项目和复杂中间件场景

**[20秒收尾]** SOC 平台告警面板状态用 Zustand 管理（轻量），全局认证状态用 Context，跨模块大状态（如告警过滤器配置）用 Redux Toolkit。

### A5: 性能优化
**[10秒结论]** React.memo 缓存组件渲染结果（浅比较 props），useMemo 缓存计算值，useCallback 缓存函数引用；懒加载用 React.lazy + Suspense 做代码分割。

**[90秒拆解]**
- **React.memo**：高阶组件，包装后组件在 props 不变时不重新渲染。默认浅比较 props，自定义比较函数做深度比较（但性能开销大）
- **useMemo/useCallback**：useMemo 缓存 `() => value` 的计算结果，useCallback 是 `useMemo(() => fn, [deps])` 的语法糖
- **Reconciliation vs Commit**：
  - Reconciliation：Fiber 阶段，计算 DOM 变化（可中断）
  - Commit：同步将变化写入 DOM，不可中断
- **懒加载**：
  ```javascript
  const Dashboard = React.lazy(() => import('./Dashboard'));
  <Suspense fallback={<Loading />}>
    <Dashboard />
  </Suspense>
  ```
- **窗口化**：对于超长列表（如 10000 行），只渲染可见区域的少量节点。`react-window` 用 FixedSizeList / VariableSizeList 实现，大幅减少 DOM 节点

**[20秒收尾]** 性能优化优先级：先测（React DevTools Profiler），再优化（避免匿名函数作为 props、减少不必要的 re-render），最后考虑 memo。

### A6: 高级模式
**[10秒结论]** Render Props 是将 render 函数作为 props 传入以复用逻辑；HOC 是接收组件返回新组件的函数；Hooks 是更现代的逻辑复用方式，无 HOC 的嵌套地狱和 ref 丢失问题。

**[90秒拆解]**
- **Render Props**：将 JSX 作为 children 或 render prop 传入，数据逻辑在父组件处理：
  ```jsx
  <MouseRenderProps render={pos => <Cat mouse={pos} />} />
  ```
- **HOC**：如 `withRouter`、`connect(mapState)`，通过 props 注入逻辑，但多层 HOC 导致 `<WithAuth(WithTheme(App))>` 嵌套地狱，且 ref 需要 forwardRef 转发
- **Hooks 优势**：无需包装组件树、无 ref 丢失问题、可在普通函数组件中复用逻辑，更符合 React 组合优于继承的设计哲学
- **Compound Components**：`Select>` + `<Option>` 隐式共享 state，无需 props drilling：
  ```jsx
  <Select>
    <Option value="a">A</Option>
  </Select>
  ```
- **forwardRef**：转发 ref 到 DOM 元素或 class 组件，React 16.3 引入。RTK 的 `connect` 内部已处理 ref 转发

**[20秒收尾]** 2020 年后新项目首选 Hooks 封装逻辑，HOC 和 Render Props 只在维护旧代码时使用。

### A7: React 18 新特性
**[10秒结论]** React 18 Concurrent Mode 让渲染可中断，useTransition 标记非紧急更新（可并发），useDeferredValue 做值防抖，Automatic Batching 自动批量处理所有状态更新。

**[90秒拆解]**
- **Concurrent Mode**：同一时间可以有多条 render 任务在内存中同时进行，优先级高的（如用户输入）可打断优先级低的（如大量列表渲染）
- **useTransition**：
  ```javascript
  const [isPending, startTransition] = useTransition();
  startTransition(() => { setQuery(input); });
  ```
  标记 setQuery 为非紧急更新，紧急更新（如输入框响应）可打断低优先级更新
- **useDeferredValue**：`const deferredQuery = useDeferredValue(query)`，query 更新时 UI 立即更新，deferredQuery 稍后更新（防抖效果）
- **Automatic Batching**：React 18 之前只有事件处理函数内的 setState 被批量，Promise/setTimeout/setInterval 内的 setState 会触发多次重渲染。18 后全部批量，减少重渲染次数
- **Strict Mode 双重调用**：开发模式下 React 故意挂载→卸载→再挂载组件，用于暴露副作用清理不完整的问题

**[20秒收尾]** React 18 SSR 的 Suspense：水合（hydration）可以并发进行，服务器流式 HTML + 客户端选择性水合，用户体验大幅提升。

### A8: TypeScript 集成
**[10秒结论]** React.FC + 接口定义 props 类型；useRef 泛型填 DOM 元素类型或 null；事件处理用 React.ChangeEvent/React.MouseEvent 等；forwardRef 泛型通过 React.ForwardRefRenderFunction 保留。

**[90秒拆解]**
- **props 类型定义**：
  ```typescript
  interface ButtonProps {
    label: string;
    onClick: () => void;
    variant?: 'primary' | 'secondary';
  }
  const Button: React.FC<ButtonProps> = ({ label, onClick, variant = 'primary' }) => ...
  ```
- **useRef 泛型**：
  ```typescript
  const inputRef = useRef<HTMLInputElement>(null);  // DOM ref
  const timerRef = useRef<number | null>(null);    // 可变值 ref
  ```
- **事件处理函数**：
  ```typescript
  const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => { ... }
  const handleClick = (e: React.MouseEvent<HTMLButtonElement>) => { ... }
  ```
- **forwardRef 保留泛型**：
  ```typescript
  interface Props { label: string; }
  const FancyButton = React.forwardRef<HTMLButtonElement, Props>((props, ref) => (
    <button ref={ref}>{props.label}</button>
  ));
  ```
- **React.ComponentProps**：提取组件的 props 类型（不含 ref）；React.ElementType 获取组件的渲染类型

**[20秒收尾]** 用 `ComponentProps<typeof MyComponent>` 从已有组件反推 props 类型，避免重复定义。
