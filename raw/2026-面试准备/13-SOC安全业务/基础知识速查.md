## 细节

### [初] 安全运营基础

**SOC 团队角色**
- 安全分析师：监控告警，做初步分析（Tier 1/Tier 2）
- 安全工程师：维护安全工具、编写检测规则
- 安全运营经理：协调响应、管理 SLA

**告警生命周期**
```
产生（SIEM）→ 分诊（AI/LLM）→ 分析（分析师）
→ 调查 → 响应（SOAR/人工）→ 关闭
```

**MTTD / MTTR 基准**（2024 Verizon DBIR）
- MTTD 业界均值：~200 天（APT 检测）
- MTTR 业界均值：~70 天
- 领先水平（<25th percentile）：MTTD < 24h，MTTR < 30 天

---

### [初] 威胁情报

**IOC 类型**
- IP：C2 服务器、僵尸网络节点
- 域名：钓鱼 URL、DGA 生成域名
- Hash：PE Hash（MD5/SHA1/SHA256）
- URL：社工钓鱼 URL、C2 通信路径

**IOC 生命周期**：生成 → 分发（STIX/TAXII）→ 匹配 → 过时（清理）

**STIX/TAXII**
- STIX 2.1：结构化描述 IOC（JSON 格式）
- TAXII 2.1：传输协议（PKI 认证 HTTPS API）

### [中] 告警降噪

**规则调优**：对触发频率高但真告警率 < 5% 的规则收紧条件

**ML 分类**：将告警上下文打包为特征向量，置信度 < 0.6 不推送

**Feedback 闭环**：分析师处置结果（确认/降级/升级）回流为训练数据，月度评估

**误报率定义**："被判定为无需处置的告警" / "总推送告警"

---

### [中] 横向移动检测

**NDR 路线（网络流量）**
- 检测 SMB/PsExec/WMI 远程执行特征
- 同一源 IP 对内网多主机 445/3389 端口连接
- 工具：Zeek + Suricata + SIEM 关联

**EDR 路线（主机日志）**
- Windows 4688 Process Create 异常链（excel→cmd→powershell）
- Windows 4624/4625 同一账户短时间多主机登录
- Linux auditd：SSH 密钥复用

**联动检测**：NDR 发现横向路径图 + EDR 确认可疑进程，置信度远高于单一数据源

---

### [高] APT 检测

**低慢大攻击特征**
- 低：低频低熵（单次行为不触发阈值）
- 慢：长时间跨度（数周至数月）
- 大：大规模目标

**检测策略**
- 长周期行为基线：90 天用户/资产行为基线，偏离 3σ 触发告警
- 多粒度滑动窗口：短窗口（1h）低阈值 + 长窗口（30 天）低阈值，两路并行
- 威胁狩猎（Threat Hunting）：假设驱动调查，主动发现盲区异常

---

### [高] 合规审计

**等保三级要求**
- 审计日志保留 ≥ 5 年
- 操作日志保留 ≥ 5 年
- 必须防篡改（WORM）
- 必须有备份

**防篡改机制**
- WORM（Write Once Read Many）存储
- SHA256 哈希链：每条日志附带 hash，下游可校验完整性

**多租户隔离**
- 数据库层按 tenant_id 行级隔离
- BI 工具 row-level security 实现逻辑隔离
- 网络层 VPC/安全组隔离
