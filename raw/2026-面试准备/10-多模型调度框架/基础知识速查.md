## 细节

### [初] 模型调用基础

**OpenAI vs Claude vs Ollama 响应格式**

| 模型 | 响应路径 | 特殊处理 |
|------|---------|---------|
| OpenAI | `resp.choices[0].message.content` | 标准 |
| Claude | `resp.content[0].text` | 响应路径不同 |
| Ollama | `resp["message"]["content"]` | usage=0 需估算 |

**Token 费用参考（2025年）**

| 模型 | 输入 $/1M tokens | 输出 $/1M tokens |
|------|-----------------|-----------------|
| GPT-4o | $2.5 | $10 |
| GPT-4o-mini | $0.15 | $0.60 |
| qwen-plus | $0.60 | $1.20 |
| qwen2.5:7B (Ollama) | ~$0 | ~$0 |

**Temperature**

| Temperature | 效果 | 适用场景 |
|-------------|------|---------|
| 0.0-0.3 | 确定性高 | 分类、提取、格式化 |
| 0.5-0.7 | 平衡 | 翻译、摘要 |
| 0.8-1.0 | 创意性强 | 写故事、头脑风暴 |

### [中] 模型抽象层

**统一响应格式**
```python
@dataclass
class LLMResponse:
    content: str
    model: str
    usage: dict  # {prompt_tokens, completion_tokens, total_tokens}
    finish_reason: str
    raw_response: Any = None
```

**抽象基类**
```python
class BaseLLMClient(ABC):
    @abstractmethod
    async def chat(self, messages, temperature=0.7, max_tokens=None) -> LLMResponse:
        pass
```

**Tag 定义**
| Tag | 典型模型 | 适用场景 |

candidates = [cfg for cfg in all_configs

return max(candidates, key=lambda x: x.priority).client

### [中] Fallback 降级链

**降级顺序原则**：从成本低到成本高排列（cheap → fast → powerful）

**重试与退避**
```python
for attempt in range(retry_times):
    try:
        return await client.chat(messages, **kwargs)
    except Exception:
        await asyncio.sleep(0.5 * (2 ** attempt))  # 指数退避
```

**最多 3 级**：主模型 + 2 个备选，避免链过长导致超时

from pydantic import BaseModel, ValidationError

class TriageOutput(BaseModel):

### [高] 成本优化

**Redis 语义缓存**
```python
def _hash(messages, temperature):
    payload = json.dumps({"messages": messages, "temperature": temperature}, sort_keys=True)
    return f"llm:cache:{hashlib.sha256(payload.encode()).hexdigest()}"
# 命中即返回，成本为 0
```

**Prompt 压缩**
- 保留 system prompt
- 保留最近 N 条对话（按 token 预算截断）
- 估算：中文按字符数，英文按单词数 × 1.3

**成本监控字段**：timestamp / model / mode / prompt_tokens / completion_tokens / cost_usd / latency_ms / cached
