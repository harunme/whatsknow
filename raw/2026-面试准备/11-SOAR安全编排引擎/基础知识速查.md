## 细节

### [初] SOAR 基础

**SOAR（Security Orchestration, Automation and Response）**
- 驱动跨工具自动化响应的编排引擎
- 消费 SIEM/EDR 的告警，执行封禁/隔离/通知等动作

**SOAR vs SIEM vs EDR**
- SIEM：收集日志、关联分析、生成告警，**不执行动作**
- EDR：终端检测和响应，提供 API 隔离主机、杀进程
- SOAR：消费告警，驱动跨工具自动化响应

**剧本本质**：有向状态图，节点=操作步骤，边=条件转移逻辑

---

### [初] 安全事件基础

**告警生命周期**
```
产生 → 分诊（Triage）→ 分析（Analysis）
→ 调查（Investigation）→ 响应（Response）→ 关闭（Closure）
```

**告警 P0-P3 定级**

| 级别 | 场景 | SLA |
|------|------|-----|
| P0 | 勒索软件、高管钓鱼 | 15 分钟 |
| P1 | 横向移动、已确认入侵 | 1 小时 |
| P2 | 可疑异常 | 4 小时 |
| P3 | 政策违规、扫描行为 | 24 小时 |

**MTTD / MTTR**
- MTTD：平均检测时间（Mean Time to Detect）
- MTTR：平均响应时间（Mean Time to Respond）

### [中] DSL 设计

**三种方案对比**

| 方案 | 优势 | 劣势 |
|------|------|------|
| JSON Schema | 前端易解析 | 表达力弱 |
| YAML | 人类可读、支持注释 | 复杂类型验证弱 |
| Python Class | 表达力最强 | 非工程师不友好 |

**推荐：YAML + Python Class 双轨**
- 安全分析师编辑 YAML（声明意图）
- 引擎将 YAML 编译为 Python Class（执行逻辑）

**YAML DSL 示例**
```yaml
name: phishing_lockdown
nodes:
  - id: extract_ioc
    type: action
    action: extract_ioc
  - id: human_approve
    type: approval
    timeout: 3600
    approver_role: soc_analyst
```

### [中] 核心机制

**状态机节点类型**
- `action`：执行动作（API 调用）
- `approval`：人工审批（挂起状态机）
- `condition`：条件判断（路由选择）
- `notification`：通知（Slack/邮件）

**审批超时策略**
- escalate（默认）：升级通知 SOC Lead，延后 1h
- auto_reject：自动拒绝
- auto_approve：自动审批（需 CISO 审批才可启用）

**ResourceLock（防竞态）**
```python
key = f"ip:{ip_address}"
lock = self._locks[key]
await asyncio.wait_for(lock.acquire(), timeout=30)
self._owners[key] = playbook_instance_id  # 记录持锁者
```

### [高] 回滚机制

**三层回滚**
1. 可逆操作（解封 IP）：精确回滚
2. 不可逆操作（杀进程）：补偿操作（创建快照）
3. 无法处理：升级人工

**幂等设计**：封锁 IP 重复执行不产生额外影响

### [高] 质量保障

**自动化测试**
```python
@pytest.mark.asyncio
async def test_block_domain_called(mock_dependencies):
    playbook = PhishingLockdownPlaybook(dependencies=mock_dependencies)
    state = await playbook.execute(alert)
    mock_dependencies["mail_gateway"].block_domain.assert_called_once()
```

**灰度发布**：影子模式运行 48h，对比影子执行结果与预期，达标后全量

**关键质量指标**
- 执行成功率 > 98%
- 平均执行时间 < 10 分钟（P0 告警）
- 回滚触发频率 < 2%
- 人工审批响应时间 < 30 分钟
