## 答案

### A1: SOAR 核心概念
**[10秒结论]** SOAR（Security Orchestration, Automation and Response）是安全响应的编排引擎，消费 SIEM/EDR 的告警驱动自动化动作；剧本本质是有状态的状态机，支持条件分支、人工审批、失败回滚。

**[90秒拆解]**
- **与 SIEM/EDR 的关系**：
  - SIEM：收集日志、关联分析、生成告警，**不执行动作**
  - EDR：终端检测和响应，提供 API 隔离主机、杀进程
  - SOAR：消费告警，驱动跨工具自动化响应
- **为什么需要 SOAR**：
  - 脚本是线性执行，无状态记忆，无法处理人工审批、失败回滚
  - SOAR 是有向状态图，节点=操作步骤，边=条件转移
  - 支持并行剧本、锁机制、超时管理、审计日志
- **本质**：有状态的状态机，LangGraph 提供开箱即用的条件路由和 checkpoint

**[20秒收尾]** SIEM 产生告警 → SOAR 自动响应，两者协同，缺一不可。SOAR 是安全运营的"中枢神经"。

---

### A2: DSL 设计
**[10秒结论]** DSL 将剧本意图（YAML）和执行逻辑（Python）解耦；推荐 YAML（分析师友好）+ Python Class（开发者可扩展）双轨制；ReactFlow 可视化生成 YAML，代码编辑器修改 YAML 同步刷新图形。

**[90秒拆解]**
- **三种方案对比**：
  - JSON Schema：易前端解析，无缝 REST API，但表达力弱
  - YAML：人类可读，支持注释，适合配置文件
  - Python Class：表达力最强，可复用逻辑片段
- **推荐双轨制**：安全分析师编辑 YAML（声明意图）→ 引擎编译为 Python Class（执行逻辑）→ 中间 AST 解耦"说什么"和"怎么做"
- **DSL 示例**：
  ```yaml
  nodes:
    - id: extract_ioc
      type: action
      action: extract_ioc
    - id: human_approve
      type: approval
      timeout: 3600
      approver_role: soc_analyst
  ```

**[20秒收尾]** DSL 是 SOAR 平台的核心抽象，设计好 DSL 就成功了一半；好的 DSL 让安全分析师无需写代码也能构建剧本。

---

### A3: 人工审批节点
**[10秒结论]** 审批节点将状态机挂起（status=paused），写入 DB 等待人工决策；超时策略默认为 escalate（升级），auto_approve 只用于低风险非破坏性操作。

**[90秒拆解]**
- **挂起机制**：`approval_node` 设置 `status: paused`，审批请求写入 `pending_approvals` 表，前端展示待审批列表
- **超时策略**：
  - escalate（升级）：通知 SOC Lead，延后 1h（默认）
  - auto_reject：自动拒绝（低风险通知类）
  - auto_approve：自动审批（需 CISO 审批才可启用）
- **失联处理**：超过 2 次升级通知无响应，剧本进入人工介入模式（暂停 + 告警 SOC Lead）
- **auto_approve 风险**：高危操作（隔离主机、封锁用户）禁止 auto_approve，只能 escalate

**[20秒收尾]** 审批节点是 SOAR 和纯脚本的本质区别——允许人工介入判断，是安全合规的基本要求。

---

### A4: 执行上下文隔离
**[10秒结论]** 通过 `ResourceLockManager` 实现资源级互斥锁，以 `resource_type:resource_id` 为键，`asyncio.Lock` 控制并发访问；TTL 防死锁，生产环境需 Redis 分布式锁。

**[90秒拆解]**
- **锁实现**：
  ```python
  key = f"ip:{ip_address}"
  lock = self._locks[key]
  await asyncio.wait_for(lock.acquire(), timeout=30)
  self._owners[key] = playbook_instance_id  # 记录持锁者
  ```
- **TTL 设置**：`2x 任务预估执行时间`，最少 30 分钟；防止持锁进程崩溃后锁永不释放
- **分布式锁**：生产环境用 Redis `SET key val NX EX ttl`，多副本 SOAR 引擎共享锁状态
- **锁超时处理**：抛出 `ResourceLockTimeout`，外部捕获后可重试或降级

**[20秒收尾]** 资源锁是防止多剧本并发冲突的核心机制，无锁则可能出现剧本 A 封禁 IP 1.2.3.4 而剧本 B 同时解封 1.2.3.4 的竞态。

---

### A5: 执行超时机制
**[10秒结论]** 三层超时：节点超时（60s，重试→失败）、审批超时（3600s，升级/自动审批/拒绝）、剧本超时（7200s，中止+告警）；超时通过后台协程监控。

**[90秒拆解]**
- **节点级超时**：
  ```python
  result = await asyncio.wait_for(
      action.execute(),
      timeout=60
  )
  ```
  指数退避重试（最多 3 次），全部超时则标记节点失败
- **审批超时**：超时刻度 `deadline` → 升级通知（默认）；超时刻度+1h → 判定失联 → 暂停 + 告警
- **剧本超时**：后台协程监控 `deadline`，超时则中止剧本 + 通知 SOC Lead + 生成事故报告
- **告警**：剧本超时/失败是 P1 告警，需立即人工介入

**[20秒收尾]** 超时机制保证系统不会因单个剧本卡死而影响整体响应，SLA 目标：P0 告警的 SOAR 剧本平均执行时间 < 10 分钟。

---

### A6: 50+ 标准剧本
**[10秒结论]** 50+ 是 SOAR 剧本数量（与 MITRE 战术技术无关）；剧本以规则+脚本执行为主，LLM 用于告警摘要和攻击阶段推理；回滚通过幂等操作+补偿机制+升级兜底三层保证。

**[90秒拆解]**
- **剧本分类**：
  - 钓鱼封禁：IOC 提取 + 邮件网关封锁 + 主机隔离审批
  - 横向移动阻断：EDR API + 风险评分 + 自动/审批封禁
  - 挖矿检测：CPU+矿池IP+进程名三合一确认 → 置信度≥0.9 自动杀进程
  - 勒索软件：快照备份 + 隔离主机 + 创建事故
- **LLM 角色**：告警上下文摘要、MITRE ATT&CK 阶段推理；核心动作执行走确定性脚本
- **回滚三层**：
  - 可逆操作（封禁/解封）：精确回滚
  - 不可逆操作（杀进程）：补偿操作（创建快照）
  - 无法处理：升级人工

**[20秒收尾]** SOAR 剧本的本质是"有人判断+自动化执行"，LLM 负责判断，脚本负责执行。

### A7: 可视化编排实现
**[10秒结论]** ReactFlow 渲染四种节点类型（Action/Approval/Condition/Notification），拖拽操作实时生成 YAML DSL，YAML 修改通过解析器同步刷新图形节点位置。

**[90秒拆解]**
- **节点类型映射**：
  - ActionNode：绿色圆角矩形，执行动作
  - ApprovalNode：黄色菱形，人工审批
  - ConditionNode：灰色六边形，条件判断
  - NotificationNode：蓝色圆形，通知
- **双向同步**：拖拽 → 更新节点坐标 → 生成 YAML；代码编辑器修改 YAML → 解析 AST → 更新节点图
- **ReactFlow 集成**：每个节点携带 `data`（DSL schema），`onNodesChange` 监听拖拽事件，同步更新本地 DSL 状态
- **DSL 解析**：YAML → AST（抽象语法树）→ LangGraph 节点注册，无缝衔接可视化与代码

**[20秒收尾]** 可视化编辑器让安全分析师无需写代码也能构建剧本，是 SOAR 平台降低使用门槛的关键功能。

---

### A8: 质量保障
**[10秒结论]** 目标：剧本执行成功率 > 98%，回滚触发频率 < 2%；通过 pytest 自动化测试 + 影子模式灰度 + 监控指标持续保障。

**[90秒拆解]**
- **质量指标**：
  - 执行成功率 > 98%
  - 平均执行时间 < 10 分钟（P0 告警）
  - 人工审批响应时间 < 30 分钟
  - 回滚触发频率 < 2%
- **自动化测试**：
  - Mock 所有外部 API（EDR、邮件网关）
  - 测试边界条件（审批拒绝、超时、并发冲突）
  - YAML Schema 校验（JSON Schema 验证 + 死节点检测）
- **灰度发布**：先在非生产环境运行 48h 影子模式，对比影子执行与预期结果，达标后全量切换
- **监控告警**：执行失败率、审批超时率、回滚触发率设阈值告警

**[20秒收尾]** 剧本测试和代码测试同等重要，用 mock 测试保证逻辑正确，用影子模式保证上线安全。
