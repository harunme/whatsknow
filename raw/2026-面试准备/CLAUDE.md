# CLAUDE.md - 面试准备目录

## 目录用途
本目录专用于 AI/工程/架构等技术面试的系统性复习。

## 文件夹结构

```
知识点索引.md
/知识点1文件夹/
  面试题.md
  基础知识速查.md
/知识点2文件夹/
  面试题.md
  基础知识速查.md
```

## 面试题.md 格式规范

### 结构要求
1. 所有面试题 + 所有面试答案，序号严格一一对应
2. 先罗列全部面试题（每题附追问），再统一给出全部答案

### 面试题部分
```markdown
## 面试题

### Q1: [主题问题]
**追问：**
- [追问1]
- [追问2]

### Q2: [下一个问题]
...
```

### 面试答案部分
```markdown
## 答案

### A1: [对应Q1]
**[10秒结论]** 一句话核心答案（结论先行）

**[90秒拆解]** 分维度展开：
- 维度1：...
- 维度2：...
- 维度3：...

**[20秒收尾]** 结合实际场景/项目经验收尾

### A2: [对应Q2]
...
```

### 答题时间分配
- 结论先行：10秒内给出核心答案
- 维度拆解：90秒内分2-4个维度展开
- 场景收尾：20秒内结合真实项目/踩坑经验

---

## 基础知识速查.md 格式规范

### 结构要求
1. 先给出大纲（知识框架）
2. 再给出细节展开

### 大纲部分
```markdown
## 大纲

- [初] 知识点A：简述
- [中] 知识点B：简述
- [高] 知识点C：简述
```

### 细节部分
```markdown
## 细节

### [初] 知识点A
详细内容...

### [中] 知识点B
详细内容...

### [高] 知识点C
详细内容...
```

### 等级标注
- **[初]** 初级：面试必问、必须秒答的基础概念
- **[中]** 中级：能讲清楚原理和 trade-off
- **[高]** 高级：能结合场景、深度优化、面试加分项

---

## 生成新知识点文件夹时的行为

1. 先创建文件夹
2. 同时生成 `面试题.md` 和 `基础知识速查.md`
3. 面试题数量建议 5-10 道，涵盖初/中/高级
4. 基础知识的深度根据等级调整

## 注意事项

- 只回答面试相关问题
- 保持文件格式一致性
- 鼓励举真实项目例子
