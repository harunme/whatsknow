## 学习笔记.md 格式规范

### 结构要求
1. 先给目录（跳转链接）
2. 再分章节详细讲解
3. 目录和正文章节标题使用中文序号

### 格式示例
```markdown
# Python 学习笔记

> 简短说明笔记面向谁、目标是什么。

---

## 目录

[一、基础语法与数据类型](#一、基础语法与数据类型)
[二、控制流与函数](#二、控制流与函数)
[三、数据结构](#三、数据结构)

---

## 一、基础语法与数据类型

### 1.1 第一个 Python 程序

代码示例...

---

## 二、控制流与函数

### 2.1 if 语句
...
```

### 规则
- **目录链接**：`[一、标题](#一、标题)` — 中文序号 + 标准 Markdown 锚点
- **正文章节**：`## 一、标题` — 与锚点一一对应
- **序号映射**：一（1）、二（2）、三（3）...十一（11）、十二（12）
- 目录中的 `---` 分隔线放在最后一个目录项之后、正文章节之前
- Obsidian 和通用 Markdown 渲染器均可正常跳转，无需 wiki-link

## 基础知识速查.md 格式规范

### 结构要求
1. 先给大纲（知识框架，含可点击链接）
2. 再给细节展开

### 大纲格式（wiki-link 锚点）
```markdown
## 大纲

- [[初]基础数据类型](#[初]基础数据类型)：int/float/str/bool/list/dict/set/tuple、不可变类型、is vs ==
- [[中]协程](#[中]协程)：asyncio 事件循环、async/await、gather vs create_task
- [[高]内存管理](#[高]内存管理)：引用计数、GC 分代回收、内存泄漏、weakref
```

### 细节格式
```markdown
## 细节

### [初] 基础数据类型

详细内容...

### [中] 协程

详细内容...
```

### 规则
- 大纲链接：`[[初]标题](#[初]标题)` — wiki-link 格式，无空格，链接指向 `### [初] 标题`
- 大纲描述跟在链接后面的 `)：` 之后（如 `[[初]基础数据类型](#[初]基础数据类型)：描述内容`）
- 细节标题：`### [初] 标题` — 不加数字序号
- 等级标注：**[初]** 初级、**[中]** 中级、**[高]** 高级

## 生成新知识点文件夹时的行为

1. 先创建文件夹
2. 同时生成 `学习笔记.md`、`面试题.md` 和 `基础知识速查.md`
3. 面试题数量建议 5-10 道，涵盖初/中/高级
4. 基础知识的深度根据等级调整

## 注意事项

- 只回答面试相关问题
- 保持文件格式一致性
- 鼓励举真实项目例子
