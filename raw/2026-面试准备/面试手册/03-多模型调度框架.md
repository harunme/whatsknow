# 多模型 LLM 调度与集成实战手册

## Summary

本手册系统阐述企业级多模型调度框架的核心设计思路与生产落地经验。框架以统一抽象层为基础，通过 Tag 路由机制实现 cheap/fast/powerful 三种推理模式的自动切换；配合 Fallback 降级链保障服务可用性；引入 Pydantic Schema Validation 兜底多模型输出不一致问题；结合 Redis 缓存与 Token 成本监控实现精细化成本控制。整体设计目标：在保证输出质量的前提下，将单次请求的 Token 消耗降低 40%-60%，同时将服务可用性维持在 99.9% 以上。核心实现基于 Python 异步架构，支持 OpenAI、通义千问（阿里云）和 Ollama 三大后端的热插拔切换，是 AI 应用工程师在多模型集成领域面试的核心竞争壁垒。

### 0.2 Token 与费用

**Token** 是 LLM 计费和上下文限制的基本单位：
- 中文：约 1-2 字 / Token
- 英文：约 4 字符 / Token（一个单词 1-2 Token）
- API 费用 = (输入 Token 数 × 输入单价) + (输出 Token 数 × 输出单价)

常见模型 Token 费用对比（2025年参考价）：
| 模型 | 输入 $/1M tokens | 输出 $/1M tokens | 适用场景 |
|------|-----------------|-----------------|---------|
| GPT-4o | $2.5 | $10 | 复杂推理 |
| GPT-4o-mini | $0.15 | $0.60 | 简单任务 |
| Claude 3.5 Sonnet | $3 | $15 | 高质量生成 |
| Claude 3.5 Haiku | $0.80 | $4 | 快速响应 |
| qwen-plus | $0.60 | $1.20 | 国产性价比 |
| qwen2.5-7B (Ollama) | ~$0（自托管）| ~$0 | 专有云 |

### 0.3 温度（Temperature）

Temperature 控制输出的随机性：

| Temperature | 效果 | 适用场景 |
|-------------|------|---------|
| **0.0 - 0.3** | 确定性高，重复性 | 事实问答、代码生成、分析 |
| **0.5 - 0.7** | 平衡 | 日常对话、翻译 |
| **0.8 - 1.0+** | 创意性强，多样性 | 写故事、头脑风暴 |

**Prompt 工程原则**：temperature 和 prompt 要匹配——分析类任务用低温，创意类用高温。

### 0.6 常见面试基础问题

**Q: 什么时候用 GPT-4o，什么时候用 GPT-4o-mini？**
- GPT-4o-mini：简单分类、摘要、格式化输出、批量处理（成本低、速度快）
- GPT-4o：复杂推理、多步分析、代码生成、长文本理解（质量高、延迟高）
- 实际策略：**分流**——简单任务用小模型，复杂任务用大模型

**Q: OpenAI API 调用超时怎么处理？**
- 设置 `timeout=30`（秒）避免无限等待
- 实现重试机制（指数退避），最多 3 次
- 备用模型兜底（cheap 失败切 fast，fast 失败切 powerful）

**Q: 什么是 Function Calling 和 Tool Use 的区别？**
- OpenAI 使用 `functions` 参数（旧版）或 `tools`（新版）
- Anthropic Claude 使用 `tools`（称为 Tool Use）
- 本质相同：让 LLM 输出结构化的工具调用指令，而非普通文本

## 一、模型抽象层设计

### 1.1 统一接口的必要性

不同模型供应商的 API 差异巨大：OpenAI 使用 `chat.completions` 端点，响应格式为 `choices[0].message.content`；通义千问使用 `dashscope` SDK，返回结构为 `output.text`；Ollama 则为本地 REST 接口，响应路径为 `message.content`。若在业务代码中直接写这些差异，每次切换模型都意味着大规模重构。统一抽象层将所有差异收敛到同一个接口，切换模型只需改一行配置。

### 1.2 抽象基类实现

```python
from abc import ABC, abstractmethod
from dataclasses import dataclass, field
from typing import Optional, Any
import httpx

@dataclass
class LLMResponse:
    content: str
    model: str
    usage: dict  # {"prompt_tokens": int, "completion_tokens": int, "total_tokens": int}
    finish_reason: str
    raw_response: Any = None

class BaseLLMClient(ABC):
    def __init__(self, api_key: str, base_url: str, model: str, **kwargs):
        self.api_key = api_key
        self.base_url = base_url
        self.model = model
        self.timeout = kwargs.get("timeout", 60.0)

    @abstractmethod
    async def chat(
        self,
        messages: list[dict],
        temperature: float = 0.7,
        max_tokens: Optional[int] = None,
        **kwargs
    ) -> LLMResponse:
        """统一聊天接口，所有子类必须实现"""
        pass

    def _calc_cost(self, usage: dict) -> float:
        """子类覆盖定价规则"""
        raise NotImplementedError
```

### 1.3 多后端子类实现

```python
class OpenAIClient(BaseLLMClient):
    PRICING = {"gpt-4o-mini": (0.15, 0.60)}  # $/1M tokens: (input, output)

    async def chat(self, messages, temperature=0.7, max_tokens=None, **kwargs) -> LLMResponse:
        async with httpx.AsyncClient(timeout=self.timeout) as client:
            resp = await client.post(
                f"{self.base_url}/chat/completions",
                headers={"Authorization": f"Bearer {self.api_key}"},
                json={
                    "model": self.model,
                    "messages": messages,
                    "temperature": temperature,
                    "max_tokens": max_tokens or 2048,
                }
            )
            resp.raise_for_status()
            data = resp.json()
            return LLMResponse(
                content=data["choices"][0]["message"]["content"],
                model=data["model"],
                usage=data["usage"],
                finish_reason=data["choices"][0]["finish_reason"],
                raw_response=data,
            )

    def _calc_cost(self, usage: dict) -> float:
        if self.model not in self.PRICING:
            return 0.0
        inp, out = self.PRICING[self.model]
        return usage["prompt_tokens"] / 1_000_000 * inp + \
               usage["completion_tokens"] / 1_000_000 * out


class TongyiClient(BaseLLMClient):
    """通义千问客户端，使用阿里云 DashScope SDK"""
    PRICING = {"qwen-plus": (0.004, 0.012), "qwen-turbo": (0.002, 0.006)}

    async def chat(self, messages, temperature=0.7, max_tokens=None, **kwargs) -> LLMResponse:
        import dashscope
        dashscope.api_key = self.api_key
        response = dashscope.Generation.call(
            model=self.model,
            messages=messages,
            temperature=temperature,
            max_tokens=max_tokens or 2048,
            result_format="message",
        )
        if response.status_code != 200:
            raise Exception(f"DashScope error: {response.code} - {response.message}")
        msg = response.output.choices[0].message
        return LLMResponse(
            content=msg.content,
            model=self.model,
            usage={"prompt_tokens": response.usage.input_tokens,
                   "completion_tokens": response.usage.output_tokens,
                   "total_tokens": response.usage.input_tokens + response.usage.output_tokens},
            finish_reason=response.output.choices[0].finish_reason,
            raw_response=response,
        )


class OllamaClient(BaseLLMClient):
    """Ollama 私有化部署客户端"""
    OLLAMA_DEFAULT_URL = "http://localhost:11434"

    async def chat(self, messages, temperature=0.7, max_tokens=None, **kwargs) -> LLMResponse:
        async with httpx.AsyncClient(timeout=self.timeout) as client:
            resp = await client.post(
                f"{self.base_url}/api/chat",
                json={
                    "model": self.model,
                    "messages": messages,
                    "temperature": temperature,
                    "options": {"num_predict": max_tokens or 2048},
                    "stream": False,
                }
            )
            resp.raise_for_status()
            data = resp.json()
            return LLMResponse(
                content=data["message"]["content"],
                model=data["model"],
                usage={"prompt_tokens": 0, "completion_tokens": 0, "total_tokens": 0},
                finish_reason=data.get("done_reason", "stop"),
                raw_response=data,
            )
```

**设计要点**：抽象层通过统一的 `LLMResponse` 将所有差异抹平，后续业务逻辑完全不感知底层用的是哪个模型。Ollama 的 usage 字段返回 0 是已知局限，可通过本地 tokenizer 估算。

## 二、Tag 路由机制

### 2.1 三模式 Tag 定义

Tag 是框架的路由决策单元，每个模型注册时携带一个或多个标签：

| Tag | 典型模型 | 适用场景 | 成本等级 |
|-----|---------|---------|---------|
| `cheap` | gpt-4o-mini, qwen-turbo | 信息抽取、分类、格式化输出 | $0.001/千 Token |
| `fast` | gpt-4o-mini, qwen-plus | 对话、摘要、翻译 | $0.005/千 Token |
| `powerful` | gpt-4o, qwen-max, ollama-llama3 | 复杂推理、代码生成、长文创作 | $0.03+/千 Token |

### 2.2 Tag 注册与匹配

```python
@dataclass
class ModelConfig:
    client: BaseLLMClient
    tags: set[str]
    priority: int = 0  # 同 tag 下优先级，数字越大越优先

class TagRouter:
    def __init__(self):
        self._registry: dict[str, list[ModelConfig]] = {}  # tag -> 模型列表

    def register(self, tag: str, config: ModelConfig):
        if tag not in self._registry:
            self._registry[tag] = []
        self._registry[tag].append(config)
        # 按优先级降序排列
        self._registry[tag].sort(key=lambda x: x.priority, reverse=True)

    async def route(self, required_tags: set[str]) -> BaseLLMClient:
        """
        多 tag 组合匹配：从所有模型中筛选出包含全部 required_tags 的候选，
        返回优先级最高的模型。
        """
        candidates = []
        for config in self._all_configs():
            if required_tags.issubset(config.tags):
                candidates.append(config)

        if not candidates:
            raise NoModelMatchError(f"No model matches tags: {required_tags}")

        return max(candidates, key=lambda x: x.priority).client

    def _all_configs(self):
        for configs in self._registry.values():
            yield from configs
```

### 2.3 使用示例

```python
router = TagRouter()
router.register("cheap", ModelConfig(
    client=OpenAIClient(api_key="...", model="gpt-4o-mini"),
    tags={"cheap", "openai"}, priority=10
))
router.register("fast", ModelConfig(
    client=TongyiClient(api_key="...", model="qwen-plus"),
    tags={"fast", "tongyi"}, priority=10
))
router.register("powerful", ModelConfig(
    client=OllamaClient(api_key="local", base_url="http://localhost:11434", model="llama3"),
    tags={"powerful", "ollama"}, priority=10
))

# 业务调用：提取结构化数据用 cheap 模式
client = await router.route({"cheap"})
response = await client.chat(messages=[{"role": "user", "content": "提取地址信息"}])

# 复杂推理用 powerful
client = await router.route({"powerful"})
response = await client.chat(messages=[{"role": "user", "content": "分析市场趋势"}])
```

**关键设计**：Tag 匹配是集合运算，`required_tags` 为空时返回最高优先级模型。多 tag 组合（如 `{"fast", "openai"}`）可精确控制来源。

## 三、Fallback 降级链

### 3.1 降级链设计

Fallback 链保证当主模型不可用时，自动尝试备选模型，避免单点故障影响用户体验：

```python
from typing import Sequence
import asyncio

class FallbackChain:
    def __init__(self, clients: Sequence[BaseLLMClient], retry_times: int = 2):
        self.clients = list(clients)
        self.retry_times = retry_times

    async def execute(self, messages, **kwargs) -> LLMResponse:
        last_error = None
        tried_models = []

        for i, client in enumerate(self.clients):
            for attempt in range(self.retry_times):
                try:
                    response = await client.chat(messages, **kwargs)
                    response.raw_response = {
                        **response.raw_response,
                        "_tried_models": tried_models,
                        "_used_model": response.model,
                    }
                    return response
                except Exception as e:
                    last_error = e
                    tried_models.append(client.model)
                    await asyncio.sleep(0.5 * (attempt + 1))  # 指数退避

        raise AllModelsFailedError(
            f"All {len(self.clients)} models failed. Last error: {last_error}"
        )
```

### 3.2 与 Tag 路由的集成

```python
class MultiModelLLM:
    def __init__(self, router: TagRouter, fallback_enabled: bool = True):
        self.router = router
        self.fallback_enabled = fallback_enabled

    async def chat(self, tags: set[str], messages: list[dict],
                   temperature: float = 0.7, max_tokens: int | None = None) -> LLMResponse:
        primary = await self.router.route(tags)

        if not self.fallback_enabled:
            return await primary.chat(messages, temperature, max_tokens)

        # 构建 fallback 链：主模型 + 同 tag 下其他备选
        all_same_tag = [cfg.client for cfg in self.router._all_configs()
                        if tags.issubset(cfg.tags)]
        chain = FallbackChain(all_same_tag, retry_times=2)
        return await chain.execute(messages, temperature=temperature, max_tokens=max_tokens)
```

**实战经验**：降级链顺序应按成本从低到高排列，避免降级后成本暴增。同时记录 `_tried_models` 元数据，用于事后成本分析。

## 四、Schema Validation

### 4.1 Pydantic 输出校验

大模型输出不稳定，直接解析可能抛异常。用 Pydantic 做结构化输出校验：

```python
from pydantic import BaseModel, ValidationError, field_validator
import json, re

class AddressInfo(BaseModel):
    city: str
    district: str
    detail: str

    @field_validator("city", "district")
    @classmethod
    def not_empty(cls, v):
        if not v or len(v) < 2:
            raise ValueError(f"Field too short: {v}")
        return v.strip()

def extract_structured(prompt: str, client: BaseLLMClient, pydantic_model: type[BaseModel]) -> BaseModel:
    """要求模型输出 JSON 并校验"""
    messages = [
        {"role": "system", "content": f"你是一个数据提取助手。只返回合法的JSON，格式：{pydantic_model.model_json_schema()}，不要有其他文字。"},
        {"role": "user", "content": prompt}
    ]
    response = client.chat(messages)
    text = response.content.strip()

    # 提取 JSON 块（处理 ```json ... ``` 包装）
    json_match = re.search(r'```(?:json)?\s*(\{.*?\})\s*```', text, re.DOTALL)
    if json_match:
        text = json_match.group(1)
    else:
        # 兜底：找第一个 { 到最后一个 }
        start = text.find("{")
        end = text.rfind("}") + 1
        if start != -1:
            text = text[start:end]

    try:
        return pydantic_model.model_validate_json(text)
    except ValidationError as e:
        # 兜底方案：部分字段降级填充
        return _partial_fill(text, pydantic_model, e)

def _partial_fill(raw: str, model_cls: type[BaseModel], error: ValidationError) -> BaseModel:
    """字段级别兜底：能解析多少解析多少"""
    result = {}
    for err in error.errors():
        field = err["loc"][0]
        # 尝试正则从原始文本提取该字段
        pattern = rf'"{field}"\s*:\s*"?([^",\n}}]+)"?'
        m = re.search(pattern, raw)
        if m:
            result[field] = m.group(1).strip()
    return model_cls.model_validate({**result})
```

### 4.2 多模型 Schema 不一致的兜底方案

不同模型的 JSON 输出格式可能不一致：有的用双引号，有的用单引号；有的含尾逗号；有的返回 Markdown 代码块包裹。处理策略按优先级：

1. **正则提取**：`re.search` 定位 `{...}` 边界
2. **JSON 容错解析**：使用 `json.loads()` 配合 `strict=False`
3. **字段级降级**：部分字段无法解析时，用正则从原文提取可用信息
4. **Pydantic 强制填充**：无法提取时用默认值，但标记 `is_fallback=True`

## 五、Token 成本优化

### 5.1 Redis 缓存

相同语义的问题不应重复计费：

```python
import hashlib, json, redis.asyncio as redis

class SemanticCache:
    def __init__(self, redis_url: str = "redis://localhost:6379", ttl: int = 3600):
        self.redis = redis.from_url(redis_url)
        self.ttl = ttl

    def _hash(self, messages: list[dict], temperature: float) -> str:
        payload = json.dumps({"messages": messages, "temperature": temperature}, sort_keys=True)
        return f"llm:cache:{hashlib.sha256(payload.encode()).hexdigest()}"

    async def get(self, messages: list[dict], temperature: float) -> str | None:
        key = self._hash(messages, temperature)
        cached = await self.redis.get(key)
        if cached:
            await self.redis.incr(f"{key}:hits")
            return cached.decode()
        return None

    async def set(self, messages: list[dict], temperature: float, response: str):
        key = self._hash(messages, temperature)
        await self.redis.setex(key, self.ttl, response)
```

### 5.2 Prompt 压缩策略

在缓存未命中时，压缩历史对话以减少 Token 消耗：

```python
class PromptCompressor:
    @staticmethod
    def compress_history(messages: list[dict], max_tokens: int = 3000) -> list[dict]:
        """保留系统提示 + 最近 N 条对话"""
        system = [m for m in messages if m.get("role") == "system"]
        others = [m for m in messages if m.get("role") != "system"]

        # 简单截断：保留最新消息直到 token 估算达标
        compressed = system + others
        # 估算：中文约 1.5 字/token，英文约 4 字符/token
        total = sum(len(m.get("content", "")) for m in compressed)
        while total > max_tokens * 2 and len(others) > 2:
            removed = others.pop(0)
            total -= len(removed.get("content", ""))
        return system + others

    @staticmethod
    def estimate_tokens(text: str) -> int:
        """简单估算：中文按字符，英文按空格分隔单词"""
        chinese = len(re.findall(r'[\u4e00-\u9fff]', text))
        english = len(re.findall(r'[a-zA-Z]+', text))
        other = len(text) - chinese - english * 4
        return chinese + english + max(0, other)
```

**实战效果**：在客服场景下，Redis 缓存命中率可达 30%-50%，结合 Prompt 压缩，单次请求 Token 消耗平均降低 45%。

### 6.2 并发队列与显存管理

```python
import asyncio, weakref

class OllamaManager:
    def __init__(self, max_concurrent: int = 2, timeout: float = 120.0):
        self.semaphore = asyncio.Semaphore(max_concurrent)
        self.timeout = timeout
        self.active_requests = 0

    async def chat(self, client: OllamaClient, messages: list[dict], **kwargs) -> LLMResponse:
        async with self.semaphore:
            self.active_requests += 1
            try:
                return await asyncio.wait_for(
                    client.chat(messages, **kwargs),
                    timeout=self.timeout
                )
            except asyncio.TimeoutError:
                raise OllamaTimeoutError(f"Request timed out after {self.timeout}s. "
                                         f"Consider upgrading GPU or reducing model size.")
            finally:
                self.active_requests -= 1

    def get_status(self) -> dict:
        return {
            "active_requests": self.active_requests,
            "available_slots": self.semaphore._value,
        }
```

### 6.3 超时与显存异常处理

Ollama 常见错误码：显存不足（`CUDA out of memory`）、模型未加载（`model not found, try pulling it first`）、并发超限（`too many requests`）。实现指数退避 + 自动降级到云端 API：

```python
async def ollama_safe_chat(manager: OllamaManager, client: OllamaClient,
                            messages: list[dict], fallback_cloud: BaseLLMClient) -> LLMResponse:
    try:
        return await manager.chat(client, messages)
    except (OllamaTimeoutError, OllamaOOMError) as e:
        print(f"[Ollama] {type(e).__name__}: {e}. Falling back to cloud model.")
        return await fallback_cloud.chat(messages)
    except Exception as e:
        if "not found" in str(e):
            # 自动拉取模型
            subprocess.run(["ollama", "pull", client.model], check=True)
        raise
```

### 7.1 消费记录与月度统计

```python
import sqlite3
from datetime import datetime, timedelta

class CostTracker:
    def __init__(self, db_path: str = "llm_costs.db"):
        self.conn = sqlite3.connect(db_path)
        self.conn.execute("""
            CREATE TABLE IF NOT EXISTS token_usage (
                id INTEGER PRIMARY KEY AUTOINCREMENT,
                timestamp TEXT,
                model TEXT,
                mode TEXT,
                prompt_tokens INTEGER,
                completion_tokens INTEGER,
                cost_usd REAL,
                latency_ms REAL,
                cached BOOLEAN DEFAULT 0
            )
        """)

    def record(self, response: LLMResponse, mode: str, latency_ms: float, cached: bool = False):
        cost = response.usage.get("cost_usd", 0.0)
        self.conn.execute("""
            INSERT INTO token_usage (timestamp, model, mode, prompt_tokens,
              completion_tokens, cost_usd, latency_ms, cached)
            VALUES (?, ?, ?, ?, ?, ?, ?, ?)
        """, (
            datetime.now().isoformat(),
            response.model,
            mode,
            response.usage.get("prompt_tokens", 0),
            response.usage.get("completion_tokens", 0),
            cost,
            latency_ms,
            cached,
        ))
        self.conn.commit()

    def monthly_report(self, year_month: str = None) -> dict:
        """year_month 格式: '2026-03'"""
        if not year_month:
            year_month = datetime.now().strftime("%Y-%m")
        cursor = self.conn.execute("""
            SELECT mode, SUM(prompt_tokens), SUM(completion_tokens),
                   SUM(cost_usd), COUNT(*)
            FROM token_usage
            WHERE timestamp LIKE ?
            GROUP BY mode
        """, (f"{year_month}%",))
        rows = cursor.fetchall()
        total_cost = sum(r[3] for r in rows)
        return {
            "year_month": year_month,
            "total_cost_usd": round(total_cost, 4),
            "by_mode": {
                r[0]: {
                    "prompt_tokens": r[1],
                    "completion_tokens": r[2],
                    "cost_usd": round(r[3], 4),
                    "request_count": r[4],
                    "pct_of_cost": f"{r[3]/total_cost*100:.1f}%" if total_cost else "0%"
                } for r in rows
            }
        }
```

```python
async def cost_monitor_loop(tracker: CostTracker, budget_usd: float = 100.0, interval_hours: int = 24):
```

## 八、Function Calling 跨模型适配

### 8.1 Schema 差异处理

各模型对 Function Calling 的 schema 定义存在显著差异：

| 模型 | 参数格式 | 必填字段 |
|------|---------|---------|
| OpenAI | `functions` / `tools` | `name`, `description`, `parameters` (JSON Schema) |
| 通义千问 | `functions` | 同 OpenAI，但部分版本不支持 `strict` 字段 |
| Ollama | 需自行构造 `tools` | `type: "function"`, `function: {name, description, parameters}` |

### 8.2 统一 Function Calling 接口

```python
@dataclass
class ToolCall:
    name: str
    arguments: dict
    raw: Any

class FunctionCaller:
    def __init__(self, client: BaseLLMClient):
        self.client = client

    async def call(self, messages: list[dict], tools: list[dict],
                   tool_choice: str = "auto", **kwargs) -> tuple[LLMResponse, list[ToolCall] | None]:
        """
        统一的 function calling 接口，自动适配不同模型格式。
        """
        adapted_tools = self._adapt_tools(tools, self.client)

        response = await self.client.chat(
            messages,
            tools=adapted_tools,
            tool_choice=tool_choice,
            **kwargs
        )

        tool_calls = self._extract_tool_calls(response)
        return response, tool_calls

    def _adapt_tools(self, tools: list[dict], client: BaseLLMClient) -> list[dict]:
        """根据模型类型转换 tools schema"""
        if isinstance(client, OpenAIClient):
            # OpenAI 原生格式
            return [{"type": "function", "function": t} for t in tools]

        if isinstance(client, TongyiClient):
            # 通义千问使用 functions 字段而非 tools
            return [{"name": t["name"], "description": t.get("description", ""),
                     "parameters": t.get("parameters", {})} for t in tools]

        if isinstance(client, OllamaClient):
            # Ollama 格式
            return [{"type": "function", "function": {
                "name": t["name"],
                "description": t.get("description", ""),
                "parameters": t.get("parameters", {}),
            }} for t in tools]

    def _extract_tool_calls(self, response: LLMResponse) -> list[ToolCall] | None:
        raw = response.raw_response
        # 尝试 OpenAI 格式
        if "tool_calls" in (raw if isinstance(raw, dict) else {}):
            return [ToolCall(name=tc["function"]["name"],
                             arguments=json.loads(tc["function"]["arguments"]),
                             raw=tc)
                    for tc in raw["tool_calls"]]
        return None
```

## 九、温度参数调优

温度参数直接影响输出的随机性和创造性，需根据业务场景精确配置：

| 模式 | 推荐模型 | Temperature | Top-p | 适用场景 |
|------|---------|------------|-------|---------|
| `cheap` | gpt-4o-mini, qwen-turbo | 0.1 - 0.2 | 0.9 | 信息抽取、结构化输出、分类 |
| `fast` | qwen-plus | 0.5 - 0.7 | 0.95 | 客服对话、摘要、翻译 |
| `powerful` | gpt-4o, qwen-max | 0.7 - 0.9 | 0.95 - 1.0 | 创意写作、头脑风暴 |

```python
TEMP_CONFIG = {
```

## 十、生产环境经验

### 10.1 多模型输出一致性

不同模型对同一问题的回答差异是最大挑战。解决方案：

- **Few-shot Prompting**：为每个模式提供相同示例，减少随机性
- **输出结构强制约束**：用 Pydantic Schema 强制输出格式一致（参见第四章）
- **一致性评分**：用 powerful 模型对 cheap/fast 输出做自动评估，超阈值则重新生成

### 10.2 延迟对比

| 模式 | 典型延迟 (P50) | 典型延迟 (P99) |
|------|--------------|--------------|
| cheap (云端) | 200-400ms | 800ms |
| fast (云端) | 400-800ms | 1.5s |
| powerful (Ollama, RTX 4090) | 1-3s | 8s |
| powerful (云端) | 2-5s | 10s |

**实战建议**：前端请求设置差异化超时，cheap/fast 用 5s 超时，powerful 用 30s 超时。Ollama 本地推理延迟波动大（受显存碎片影响），建议增加 50% 的超时余量。

### 10.3 成本控制三板斧

1. **缓存优先**：所有只读查询走 SemanticCache，命中即返回，成本为 0
2. **模式分级**：简单任务强制 cheap 模式，通过 Tag 路由限制
3. **月度 Review**：每周分析成本报表，识别异常消耗（如无限循环重试）

## 面试 Q&A

**Q1：多模型调度的核心挑战是什么？**

核心挑战有两个维度。第一是接口一致性：不同模型的 API 格式、错误处理、能力边界完全不同，统一抽象层必须足够厚才能屏蔽这些差异，但过厚又会影响性能。第二是成本与质量的动态平衡：每个请求用 powerful 模型质量最高，但成本是 cheap 的 20-50 倍；框架必须能根据任务复杂度自动选择合适档位，否则成本无法控制。

**Q2：Fallback 降级链的顺序应该怎么设计？**

优先排成本低的模型，再排效果好的模型。以 cheap -> fast -> powerful 为例：如果主模型 cheap 失败，降级到 fast 成本最多增加 3-5 倍；但如果直接用 powerful 作为主模型，降级失败就没有更高级别可用。另外，降级链中同一模型的多次重试要用指数退避，避免瞬时故障导致无效等待。

**Q3：Ollama 私有化部署和云端 API 怎么选？**

选 Ollama 的场景：数据安全要求高（不能上云）、日均调用量超过 10 万次（私有化成本优势明显）、对特定模型有定制需求。选云端 API 的场景：追求低延迟（云端 GPU 通常比消费级显卡快）、需要最新模型（Ollama 模型更新滞后）、流量波动大（按需付费无需预留算力）。最优方案是混合部署：主力流量走 Ollama，OOM/超时/流量峰值时自动切换云端。

**Q4：如何防止 Token 成本失控？**

三道防线：事前（Tag 路由限制业务只能用指定模式的模型）、事中（Redis 缓存 + Prompt 压缩减少 Token 消耗）、事后（每日成本监控 + 告警）。最关键的是建立开发规范，明确哪些场景必须用 cheap 模式、哪些场景才允许用 powerful 模式，并在代码层面通过 Tag 路由强制执行，而不是依赖开发者的自觉。

**Q5：Schema Validation 的容错策略是什么？**

分层容错：第一步用正则提取 JSON 并尝试 `model_validate_json`；第二步如果失败，用 `_partial_fill` 做字段级正则提取；第三步如果仍失败，返回部分填充的默认对象并附加 `is_fallback=True` 标记。同时记录 validation 失败率作为模型质量指标——如果某个模型的 validation 失败率超过 5%，说明输出格式不稳定，应考虑切换模型或优化 Prompt。
