## 细节

### [初] RAG 核心概念

**RAG（Retrieval-Augmented Generation）**
- LLM 的知识有截止日期且无法访问私有数据
- RAG 通过先检索相关文档，再将文档作为上下文喂给 LLM 解决这两个问题

**典型 RAG 流程**
```
用户问题 → Query Embedding → 向量数据库检索 Top-K
→ 将 K 个文档作为上下文 → LLM 生成 → 答案输出
```

**RAG vs Fine-tuning**

| 维度 | RAG | Fine-tuning |
|------|-----|-------------|
| 知识更新 | 实时可更新 | 需重新训练 |
| 答案溯源 | 可溯源 | 不可溯源 |
| 成本 | 低 | 高 |
| 适用 | 知识频繁变更 | 固定模式、高频调用 |

### [初] Embedding 与向量

**Embedding 原理**
- 将文本映射为固定长度向量
- 语义相似的文本在向量空间中距离更近
- 余弦相似度：cosine(query_vec, doc_vec) 接近 1 表示相似

**主流中文模型对比**

| 模型 | 维度 | MTEB 精度 | 特点 |
|------|------|----------|------|
| BGE-large-zh-v1.5 | 1024 | ~65.5 | 中文首选，本地部署 |
| text-embedding-3-large | 3072 | ~64.6 | OpenAI API，贵 |
| GTE-large-zh | 1024 | ~66.1 | 高精度中文 |
| Jina-embeddings-v3 | 1024 | ~67.1 | 长文本+多语言 |

**Qdrant 基本用法**
```python
client.search(collection_name="alerts", query_vector=vec, limit=5)
```

### [中] 向量数据库选型

| 数据库 | 特点 | 适用场景 |
|--------|------|---------|
| Qdrant | Rust，高性能，混合检索 | 推荐，生产环境 |
| Milvus | 分布式，超大规模 | >1 亿向量 |
| Chroma | 轻量，Python 原生 | 原型/本地开发 |
| Pinecone | 全托管，无需运维 | 快速上线 |

**HNSW 算法原理**
- 分层可导航小世界图（Hierarchical Navigable Small World）
- 搜索复杂度 O(log N)
- m：每层连接数，越大精度越高但内存越大
- ef：搜索宽度，越大精度越高但延迟越大

**HNSW 参数**
```python
hnsw_config={
    "m": 16,            # 每层连接数
    "ef_construct": 200,  # 构建时搜索宽度
}
client.search(..., params={"hnsw_ef": 128})  # 运行时搜索宽度
```

### [中] Chunking 策略

**固定窗口分块**
```python
def fixed_window(text, chunk_size=512, overlap=64):
    start = 0
    while start < len(text):
        yield text[start:start+chunk_size]
        start += chunk_size - overlap
```

**语义分块**：按段落边界切，合并至阈值上限，保证语义完整性

**chunk_size 经验值**

| 场景 | chunk_size | overlap |
|------|-----------|---------|
| SOP 文档 | 512-768 tokens | 64 tokens |
| 历史案例 | 384-512 tokens | 64 tokens |
| FAQ | 128-256 tokens | 0 |

**Trade-off**
- < 128 tokens：上下文稀缺，关系断裂
- > 1024 tokens：稀释核心信息
- 512 tokens：SOC 场景最佳平衡

### [高] 混合检索

**RRF 融合公式**
$$RRF_{doc} = \sum_{r \in R} \frac{1}{k + rank_r(doc)}$$
- k=60 是经验最优值（原始论文验证）
- k 过小排名靠后文档被过度惩罚；k 过大融合效果不明显

**双路召回**
- 向量检索：语义相似（"恶意软件"↔"病毒"）
- BM25 检索：专有名词精确匹配（CVE 编号、IP 地址）

### [高] Reranker

**交叉编码器原理**
- 向量检索（双塔模型）：query 和 doc 分别编码，距离计算
- Reranker（交叉编码器）：query + doc 配对输入，精细化相关性打分

**两阶段检索流程**
```python
# Step 1: 向量检索召回 20 个候选
candidates = client.search(collection, query_vector, limit=20)

# Step 2: Reranker 精细排序输出 top-3
scores = reranker.compute_score([[query, c.text] for c in candidates])
ranked = sorted(zip(candidates, scores), key=lambda x: x[1], reverse=True)
```

**BGE Reranker v2-m3**：`FlagReranker(model_name="BAAI/bge-reranker-v2-m3")`，本地 GPU 推理

### [高] 效果评估

**Recall@K**
$$\text{Recall}@k = \frac{|\text{relevant} \cap \text{top-k}|}{|\text{relevant}|}$$

**MRR**：第一个相关文档排名的倒数均值

**RAGAS 指标**
- faithfulness：答案对上下文的忠实度
- answer_relevancy：答案与问题的相关度
- context_precision：top contexts 的精确率

**生产目标**
- Recall@20 > 0.85
- Recall@3 > 0.70
- MRR > 0.65
- 端到端准确率（分析师采纳率）> 78%
