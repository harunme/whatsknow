## 细节

### [初] 基础数据类型

**八大类型**
- `int`（整数，无大小限制）
- `float`（双精度浮点）
- `str`（Unicode 字符串）
- `bool`（`True`/`False`，本质是 `int` 的子类）
- `list`（动态数组，可变）
- `dict`（哈希表，键需可哈希）
- `set`（无序不重复集合）
- `tuple`（不可变序列，可哈希）

**可变 vs 不可变**
- 不可变：`int/float/str/bool/tuple/frozenset`，地址不变内容不变
- 可变：`list/dict/set`，内容变化不换地址
- 函数参数传可变对象要小心——函数内部修改会影响调用者

**is vs ==**
- `is`：比较对象身份（内存地址），判断是否是同一对象
- `==`：比较值是否相等
- 小整数池（`[-5, 256]`）和字符串字面量复用对象，`is` 可能返回 True 但不应依赖
- 判断 `None` 用 `is None`（`None` 是单例）

**常见陷阱**
```python
a = [1, 2, 3]
b = a        # b 和 a 指向同一对象
b.append(4)  # a 也会变！

c = a[:]     # 浅拷贝，新列表
d = copy.deepcopy(a)  # 深拷贝
```

### [初] 函数参数

**五种参数类型**
```python
def func(a, b=1, *args, c, **kwargs):
    # a: 位置参数（必填）
    # b=1: 默认参数
    # *args: 可变位置参数（元组）
    # c: 关键字参数（必须用关键字传参）
    # **kwargs: 可变关键字参数（字典）
```

**参数顺序**：位置参数 → *args → 关键字参数 → **kwargs

**可变默认参数陷阱（常见面试坑）**
```python
def add(item, items=[]):  # ❌ 危险！默认列表在函数定义时创建一次
    items.append(item)
    return items

print(add(1))  # [1]
print(add(2))  # [1, 2]  ← 不是 [2]！

# 正确写法：
def add(item, items=None):
    items = items or []
    items.append(item)
    return items
```

**解包参数**
```python
args = (1, 2, 3)
kwargs = {"a": 4, "b": 5}
func(*args, **kwargs)  # 自动解包
```

**强制关键字参数（3.0+）**
```python
def func(a, *, b, c):  # * 之后必须用关键字传参
    pass
func(1, b=2, c=3)
```

### [初] 装饰器

**原理**：高阶函数，接收函数返回包装后的新函数

**@functools.wraps**：保留原函数 `__name__`/`__doc__` 元数据

**带参数装饰器（三层嵌套）**
```python
def retry(max_attempts=3, delay=1.0):
    def decorator(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            for i in range(max_attempts):
                try: return func(*args, **kwargs)
                except: time.sleep(delay)
            raise
        return wrapper
    return decorator
```

### [初] 生成器与迭代器

**yield vs return**
- return：结束函数返回值
- yield：暂停函数生成一个值，调用者从暂停点继续

**优势 vs 列表推导式**
- 内存占用 O(1)（列表一次性全加载）
- 适合大数据流或无限序列
- 表示状态ful 计算（斐波那契数列）

**StopIteration**：`for` 循环自动捕获；外部 `next()` 无值时抛出

### [中] 协程

**asyncio 事件循环**
- 单线程调度器，I/O 多路复用（epoll/kqueue）
- `await` 时挂起协程，切换执行其他协程

**async/await vs 同步**
- async/await 定义协程，不阻塞事件循环
- 同步代码会阻塞整个事件循环

**gather vs create_task**
- `gather`：等待全部完成返回结果列表
- `create_task`：立即返回 Task 对象，不阻塞，可精细控制取消和超时

---

### [中] 并发编程

**三者对比**

| 方案 | 特点 | 适用场景 |
|------|------|---------|
| asyncio | 协程，单线程，高并发 I/O | HTTP 请求、数据库查询 |
| threading | OS 线程，GIL 限制 | 阻塞 I/O（GIL 在 I/O 时释放）|
| multiprocessing | 进程，绕过 GIL | CPU 密集型 |

**GIL 影响**：限制同一时刻只有一个线程执行 Python 字节码；I/O 操作时 GIL 释放，threading 对 I/O 密集仍有效

### [中] 面向对象

**类变量 vs 实例变量**
- 类变量：class 体内定义，所有实例共享
- 实例变量：`__init__` 中 `self.xxx`，每个实例独立

**__new__ vs __init__**
- `__new__`：创建并返回实例对象（控制对象创建过程，如单例）
- `__init__`：初始化实例属性

**MRO**：多继承时 Python 用 C3 线性化算法决定方法解析顺序，`ClassName.__mro__` 查看

### [高] 内存管理

**引用计数**：每对象维护计数器，为 0 立即释放；循环引用（A→B→A）需 GC 模块定期检测

**GC 分代回收**
- 年轻代（频繁回收）：新创建对象
- 老年代：存活越久的对象
- 新建对象在年轻代，存活越久越可能长期存在

**常见内存泄漏**
- 全局列表无限 append
- `@lru_cache(maxsize=None)` 无上限缓存
- 未关闭文件句柄

**weakref**：`weakref.ref(obj)` 创建弱引用，不阻止 GC

### [高] Web 框架

**Flask**：微框架，适合小项目和 API 原型
**Django**：全功能框架（MVC+ORM+Admin+Auth）
**FastAPI**：API 优先，async/await 原生支持，性能接近 Node.js

**FastAPI 快的原因**
- Starlette（底层）+ Uvicorn（ASGI）异步架构
- Pydantic v2 的 `pydantic-core` 用 Rust 写验证逻辑
- `model_dump_json()` 使用 orjson 序列化，比标准库快 2-3 倍

### [高] 类型提示

```python
from typing import List, Optional, Union, Callable, TypeVar, Generic

List[int]           # Python 3.9+: list[int]
Optional[str]        # Python 3.10+: str | None
Union[int, str]     # Python 3.10+: int | str
Callable[[int, str], bool]  # 接收 (int, str) 返回 bool 的函数
TypeVar              # 泛型变量
```

**运行时有效性**：默认不生效，需用 mypy/pyright 静态检查，或 beartype 运行时验证
