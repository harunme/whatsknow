## 细节

### [初] 安全基础

**CIA 三元组**
- Confidentiality（机密性）：数据不被未授权访问
- Integrity（完整性）：数据不被篡改
- Availability（可用性）：系统和服务正常运行

**告警 vs 事件 vs 事故**
- 事件（Event）：任何发生的事情
- 告警（Alert）：需要关注的通知
- 事故（Incident）：确认的安全事件，需要应急响应

**常见攻击类型**：钓鱼邮件、暴力破解、SQL 注入、XSS、零日漏洞、APT

---

### [初] SIEM 基础

**核心功能**：日志采集 → 存储 → 搜索 → 关联分析 → 告警生成

**Splunk vs Elasticsearch**

| 维度 | Splunk | Elasticsearch |
|------|--------|-------------|
| 索引格式 | TSIDX | Lucence 倒排索引 |
| 关键字搜索 | 极快 | 快 |
| 范围查询 | 需解压匹配 | 更快 |
| 成本 | 按数据量收费（贵） | 开源免费（运维复杂） |

**主流 SIEM 产品**：Splunk、ELK（Elastic Stack）、Microsoft Sentinel、Tencent Cloud SIEM

### [中] 日志采集

**FileBeat vs Syslog**

| 维度 | FileBeat | Syslog |
|------|---------|--------|
| 部署 | Agent 部署目标主机 | 推送或中转 |
| 可靠性 | 本地缓存+ACK，断点续传 | UDP 易丢包 |
| 精度 | 精确读取偏移量 | 无法精确控制 |

**At-Least-Once 三步**
1. FileBeat 发送后等 ACK，再删除本地缓存
2. 存储端幂等写入（`_id` = hash）
3. 消费者提交 OFFSET 后才认为消息处理完成

**Kafka 作用**：削峰填谷，生产速率波动时作为缓冲区

### [中] 日志归一化

**归一化目标**：将异构日志映射为统一字段（CEF/OCSF 标准）

**字段映射示例**
```
Apache: src_ip → network.source_ip, status_code → http.status_code
AWS ALB: client_ip → network.source_ip, tls.cipher → tls.version
```

**映射规则管理**
- 版本化管理（Git），CI/CD 自动推送
- 动态归一化（JSON Schema + 自动推断）减少人工维护

### [高] ATT&CK 框架

**14 个战术阶段**（速记）
初→执→持→提→躲→凭→发→横→集→控→泄→影

**核心阶段详解**
- Initial Access：钓鱼邮件、利用漏洞、供应链攻击
- Execution：恶意脚本、LOLBAS 工具
- Persistence：注册表 Run Key、计划任务
- Lateral Movement：PsExec、WMI、RDP 横向
- Command and Control：DNS 隧道、Cobalt Strike

**ATT&CK Navigator**：JSON layer 文件导入，矩阵热力图展示检测覆盖率

### [高] NL to Query

**Few-shot Prompt 结构**
```
System: 你是一个 Splunk SPL 专家...
示例1: "查找过去1小时登录失败" → "index=auth | search action=failure"
User: {用户输入}
```

**Schema 注入**：每次推理前注入用户有权访问的 index/field 白名单，防止跨租户查询

**AST 校验**：拒绝 `|script`、`|delete` 等危险操作

**覆盖范围**：常见查询（统计/搜索）覆盖 ~80%，复杂嵌套查询 ~65%

---

### [高] 告警聚合

**分层聚合架构**
- Level 1（实时，秒级）：同主机同类型立即生成 P0/P1
- Level 2（5 分钟窗口）：同主机关联告警按 ATT&CK 阶段聚合生成 Incident
- Level 3（异步，后台）：跨主机跨战术 APT 相关告警 ML 聚类

**聚合维度**：时间窗口、攻击链关联、实体维度、语义相似性

**ML 聚类**：Sentence-BERT 嵌入计算余弦相似度，>0.85 自动归并，发现未知攻击模式
