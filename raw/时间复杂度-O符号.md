# 时间复杂度：O(1)、O(N)、O(log N)

## 来源

这是**大 O 符号（Big O Notation）**，来自计算机科学中的**算法与数据结构（Algorithm & Data Structures）**领域，用于描述算法运行时间随输入规模增长的**增速级别**。

---

## 核心含义速查

| 符号 | 名称 | 含义 | 生活比喻 |
|------|------|------|---------|
| **O(1)** | 常数时间 | 输入多少，执行时间不变 | 查字典，直接翻到那一页 |
| **O(N)** | 线性时间 | 输入翻倍，时间也翻倍 | 逐个点名 N 个人 |
| **O(log N)** | 对数时间 | 输入翻倍，时间只增一点 | 二分查找，每次砍一半 |

---

## 速度排序（从快到慢）

```
O(1)          ████████████  最快
O(log N)      ████████████  较快（N=10亿，只需约30步）
O(N)          ████████████  中等（N=10亿，需10亿步）
O(N log N)    ████████████  较慢
O(N²)         ████████████  慢
O(2^N)        ████████████  极慢，实际不可用
```

> 高中数学已学 **对数 log** 和**函数单调性**，但**大 O 符号**是大学计算机专业的标准内容。
>
> 如果你正在学数据结构 → 这就是你当前的核心知识点。

## 典型算法对照

| 复杂度 | 代表算法 |
|--------|---------|
| O(1) | 数组随机访问、哈希表查找 |
| O(log N) | 二分查找 |
| O(N) | 线性查找、遍历数组 |
| O(N log N) | 快速排序、归并排序、堆排序 |
| O(N²) | 冒泡排序、选择排序、插入排序 |
| O(2^N) | 暴力枚举子集、递归斐波那契（无记忆化） |

## 学习建议

- **先理解概念**：会用语言描述 O(N) 和 O(log N) 的区别
- **再学会推导**：能对简单代码片段分析其时间复杂度
- **最后熟练**：看到排序算法能脱口而出其复杂度
