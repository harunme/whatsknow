# 时间复杂度

算法运行时间随输入规模 n 增长的增速级别，用 Big O 符号表示。

## 核心概念

### 为什么需要 Big O

不同的算法，在输入规模增大时，速度衰减程度完全不同：
- O(1)：不管输入多少，时间不变
- O(N)：输入翻倍，时间翻倍
- O(N²)：输入翻倍，时间变成 4 倍

Big O 让我们在写代码前就能预判算法在数据量增长时的表现。

### Big O 速查表

| 复杂度 | 名称 | 输入翻倍时，时间… | 典型例子 |
|---|---|---|---|
| **O(1)** | 常数时间 | 不变 | 数组随机访问、哈希表查找 |
| **O(log N)** | 对数时间 | 只增一点 | 二分查找（每次砍一半） |
| **O(N)** | 线性时间 | 翻倍 | 线性查找、遍历数组 |
| **O(N log N)** | 线性对数时间 | 略多于翻倍 | 快速排序、归并排序、堆排序 |
| **O(N²)** | 平方时间 | 变成4倍 | 冒泡排序、选择排序、插入排序 |
| **O(2^N)** | 指数时间 | 急剧膨胀 | 暴力枚举子集、无记忆化递归斐波那契 |

### 速度排序

```
O(1)          ████ 最快
O(log N)      ████ 较快（N=10亿，只需约30步）
O(N)          ████ 中等
O(N log N)    ████ 较慢
O(N²)         ████ 慢
O(2^N)        ████ 极慢，实际不可用
```

## 与 Transformer 的关联

[[自注意力]] 的计算复杂度是 **O(n²·d)**（n = 序列长度，d = 模型维度）。这是 Transformer 的主要计算瓶颈，也是后续优化（FlashAttention、Sparse Attention）的目标。

## 空间复杂度

Big O 也用于描述空间（内存）复杂度：`O(1)` = 固定内存，`O(N)` = 随输入线性增长。
