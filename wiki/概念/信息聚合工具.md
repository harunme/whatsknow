# 信息聚合工具

用于从多个来源自动收集、过滤和分发信息的开源工具组合，是 LLM 知识库数据摄入层的重要补充。

## 核心工具

| 工具 | 核心能力 | 适用场景 |
|---|---|---|
| **BestBlogs** | 聚合 400+ 信源，大模型质量打分+摘要提取 | 需要高密度精选信息流 |
| **TrendRadar** | 35 平台热搜监控，关键词过滤，Webhook 推送 | 热点追踪，竞品监控 |
| **CloudFlare-AI-Insight-Daily** | Workers 运行，AI/科技垂直领域，多形态输出 | 每日简报，公众号/RSS 推送 |

## 技术架构对比

```
BestBlogs:
  来源(X/播客/博客) → LLM质量打分 → 摘要生成 → 外文翻译
      ↓
TrendRadar:
  35平台热搜 → 关键词过滤 → 热度重排 → Webhook推送(企微/钉钉/飞书)
      ↓
CloudFlare-AI-Insight-Daily:
  AI/科技垂直抓取 → 日报生成 → 多形态输出(网页/播客/RSS/公众号)
```

## 在 LLM 知识库中的角色

信息聚合工具位于 [[Wiki 编译]] 的上游——[[数据摄入]] 阶段。它们将散落在网络各处的信息自动汇聚到可处理的位置，再由 LLM 编译为结构化 wiki。

```
信息聚合工具（BestBlogs/TrendRadar）
        ↓ raw/ 数据自动积累
[[Wiki 编译]] → wiki/
        ↓
[[LLM 原生问答]]
```

## 局限与补充

- 这些工具解决的是"信息怎么来"，不解决"知识怎么组织"
- 聚合后的信息仍需经过 [[Wiki 编译]] 才能成为可用知识
- 与 [[辅助工具]] 的关系：信息聚合工具是 LLM 知识库的外部数据源；辅助工具是 LLM 知识库的内部扩展能力
