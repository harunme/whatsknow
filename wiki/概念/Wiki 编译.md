# Wiki 编译

LLM 将原始异构文档目录转换为结构化、相互链接的 `.md` wiki 文件的过程。

## "编译"的含义

编译不仅仅是格式转换。LLM 执行以下工作：

1. **摘要** — 每份原始文档生成一篇摘要（raw/ 文件 → 摘要文章）
2. **提取概念** — 识别跨文档的重复主题、术语和思想
3. **撰写原子文章** — 每个概念独立成篇
4. **建立双向链接** — 每篇文章链接相关主题；链接是双向的
5. **维护索引文件** — 包含所有文档的简要摘要，供 Agent 快速读取

## 输入

- 文章、论文、代码仓库、数据集、图片
- 通过 Obsidian Web Clipper 转换的网页文章（图片已下载到本地）
- `raw/` 目录下的任意 `.md` 文件

## 输出结构

```
wiki/
  摘要/          # 每份 raw 文件一篇
  概念/          # 原子主题文章
  index.md       # wiki 全局概览
```

## 持续编译

编译不是一次性 ETL 作业。随着新的 raw 数据加入，LLM 增量重编译——更新摘要、创建新的概念文章、修补双向链接。Wiki 始终保持在"已编译"状态。
