# LLM 原生问答

在不依赖传统 RAG 基础设施的情况下，使用 LLM Agent 对编译后的 wiki 回答复杂研究问题。

## 关键发现

当 wiki 达到一定规模（~100 篇文章、~40 万字）时，LLM Agent 可以通过以下方式回答深度研究问题：

1. 阅读 wiki 的索引文件和文档摘要
2. 识别最相关的文章
3. 完整阅读（或相关段落）
4. 综合给出答案

**不需要**向量数据库、chunk 策略或 embedding 流水线。

## 为什么不需要 RAG

RAG（检索增强生成）是 LLM 知识 grounding 的标准方案，但在中小规模下：

- LLM 可以自行维护**索引**和**摘要**——本质上是它自己写的"软 RAG"
- Wiki 结构（双向链接、概念文章）已经编码了相关性信息
- 向量数据库的工程复杂度可能超过其带来的收益

RAG 的必要性只在知识库超过上下文窗口限制或检索延迟敏感时才会显现。

## Agent 行为

LLM Agent 收到问题时的典型行为：
- 扫描索引 → 定位候选文章
- 阅读摘要 → 缩小范围
- 阅读全文 → 综合答案
- 可选：将答案归档回 wiki，形成新文章
