# 位置编码

为 Transformer 注入**序列中 token 位置信息**的技术。因为自注意力机制本身是**位置无关的**（对每个位置的运算是对称的），所以必须显式引入位置信息。

## 为什么需要位置编码

自注意力的计算：所有 token 之间的注意力是同时、独立计算的，不存在"第 1 个 token"、"第 5 个 token"这样的概念区分。

没有位置编码，以下两个句子在 Transformer 中会产生完全相同的表示：
- "狗咬了人"
- "人咬了狗"

## 正弦/余弦位置编码

原论文提出的方案——对每个位置 pos 和维度 i：

```
PE(pos, 2i)   = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))
```

特点：
- 周期随维度 i 变化：从 `sin(2π·pos/10000)` 到 `sin(2π·pos/10000^0.5)` 到 `sin(2π·pos)`
- 不同维度的周期不同，形成唯一的位置指纹
- 线性组合可表示相对位置（物理意义）

## 可学习的位置编码

另一种常见方案：将位置编码作为可学习的参数（BERT、GPT 系列早期版本使用）。消融实验表明两种方案效果相近。

## 后续发展

| 编码方式 | 代表模型 | 特点 |
|---|---|---|
| 正弦位置编码 | 原版 Transformer | 可泛化到训练时未见过的更长序列 |
| 可学习位置编码 | BERT、GPT-2 | 更灵活，但无法外推到更长序列 |
| 旋转位置编码（RoPE） | LLaMA、GLM | 在注意力计算中融入位置信息，长上下文效果更好 |
| ALiBi（线性偏置） | BLOOM | 不添加位置编码，直接在注意力分数上加入偏置 |

## 相关概念

- [[Transformer 架构]] — 位置编码是 Transformer 的组成部分
- [[自注意力]] — 自注意力本身不包含位置信息
- [[编码器-解码器]] — 位置编码在编码器和解码器的输入处各加一份
