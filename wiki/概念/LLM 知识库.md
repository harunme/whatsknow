# LLM 知识库

一种以 LLM 为核心的个人知识库系统：LLM 不仅是检索工具，更是 wiki 的主要撰写者和维护者。

## 核心理念

传统笔记工具（Notion、Roam、Obsidian）依赖人类撰写和链接内容。而在 LLM 知识库中，人类只负责收集原材料；LLM 完成摘要撰写、概念提取、链接建立和维护更新。LLM 的 token 消耗从"操控代码"转向"操控知识"。

## 流水线

```
raw/ (原始文档)
    ↓ LLM 编译
wiki/ (摘要 + 概念文章 + 双向链接)
    ↓ LLM CLI 工具
问答 + 增强循环
    ↓ 浏览
Obsidian
```

## 关键特性

| 特性 | 说明 |
|---|---|
| **所有权** | LLM 撰写和维护 wiki；人类几乎不直接编辑 |
| **规模门槛** | 约 100 篇文章 / 40 万字以上，问答能力开始显著增强 |
| **检索方式** | 不需要复杂的 RAG——LLM 自动维护索引和摘要 |
| **累积效应** | 问答结果常被归档回 wiki，形成自我增强的知识库 |

## 为什么有效

现代 LLM 已经足够强大，能够：
- 对异构来源（文章、代码、数据集、图片）撰写连贯摘要
- 在多篇文章间保持一致的链接结构
- 发现知识缺口并建议新的文章方向
- 作为 Agent 阅读多篇文档以回答单一复杂问题
