# 编码器-解码器

Transformer 所基于的标准序列到序列（Seq2Seq）结构，包含两个主要部分：**编码器**处理输入序列，**解码器**生成输出序列。

## 编码器（Encoder）

将输入序列 `x₁, x₂, ..., x_n` 转换为连续的表示序列 `z₁, z₂, ..., z_n`：

```
输入嵌入 + 位置编码
    ↓
[编码器层 1] × N
    ↓
[编码器层 2] × N
    ...
    ↓
编码器输出（每个输入位置一个表示向量）
```

编码器的每个层级：
- **多头自注意力** → 建模输入内部的依赖关系
- **前馈网络** → 逐位置非线性变换

## 解码器（Decoder）

给定编码器输出 `z` 和已生成的目标序列 `y₁, ..., y_{t-1}`，逐步生成下一个 token `y_t`：

```
目标嵌入 + 位置编码（已生成部分）
    ↓
[解码器层 1] × N
    ↓
[解码器层 N] × N
    ↓
线性层 + Softmax → 下一个 token 的概率分布
```

解码器的每个层级：
- **掩码多头自注意力** → 建模已生成目标序列内部的依赖
- **交叉注意力** → 关注编码器输出（源序列信息）
- **前馈网络** → 逐位置非线性变换

## 掩码机制

解码器中必须**遮蔽未来 token**，防止信息泄露：

| token 位置 | 能看到的源 | 能看到的目标 |
|---|---|---|
| 位置 0 | 编码器输出 | `<start>` |
| 位置 1 | 编码器输出 | `<start>`, token₁ |
| 位置 2 | 编码器输出 | `<start>`, token₁, token₂ |
| ... | ... | ... |

## 后续变体

| 变体 | 说明 |
|---|---|
| **仅编码器**（BERT） | 适合理解任务（MLM 预训练） |
| **仅解码器**（GPT 系列） | 适合生成任务（Causal LM 预训练） |
| **编码器-解码器**（T5、BART） | 适合 seq2seq 任务（文本到文本） |
